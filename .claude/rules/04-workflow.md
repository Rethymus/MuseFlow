# 开发工作流规则

> 完整工作流步骤（功能开发 / 快速迭代 / Bug 修复 / 重构）已移至 skill `/museflow-workflow`，按需加载以省 context。本文件仅保留每轮强制的硬约束。

## 提交规范

```
feat(范围): 简短描述
fix(范围): 简短描述
refactor(范围): 简短描述
docs(范围): 简短描述
test(范围): 简短描述
chore(范围): 简短描述
```

## 禁止事项

- ❌ 没有测试就实现功能
- ❌ 一次改多个关注点
- ❌ 在测试失败时继续
- ❌ 重构和加功能混合
- ❌ 跳过 /verify 直接声称完成
