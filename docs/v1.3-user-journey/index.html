诚实边界

样例用于验证链路和效果参考，不等同于最终出版稿；真实文学质量仍需人工审读。

验证链路

世界观搭建：使用修仙模板，创建角色卡与 Skill 设定守护规则。
碎片捕捉与整理：输入灵感碎片，经过 PromptPipeline 生成连贯段落。
开篇引导：生成场景切入、人物切入、悬念切入三种开篇。
百章章节流：创建 100 章，逐章生成并验证 300–500 字输出边界。
故事结构与导出：验证伏笔生命周期、格式清洗、Markdown/TXT/JSON 三格式导出。
分析报告：呈现 token 成本、痛点、反 AI 味、知识库一致性四类报告。

如何解读这个成果站

本成果站把自动化测试中隐藏在代码和日志里的验证结果，整理为人可以直接阅读的参考材料。章节内容来自 test/journey/helpers/story_outline.dart 的确定性 100 章大纲；验证结论来自已通过的 flutter analyze 与 flutter test。

如果要看产品真实运行截图或真实模型输出，需要在后续人工 UAT 中补充 Windows/Android 设备验证、IME 体验、真实 GLM 长流程和读者盲测。
