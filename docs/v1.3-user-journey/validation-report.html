v1.3 用户视角全流程验证报告

本报告把 Phase 12–16 的自动化验证、百章修仙样例、报告中心能力和仍需人工 UAT 的边界集中呈现，供验收和后续产品决策参考。

成果首页 百章样例

验证目标

代入核心用户视角，用 MuseFlow 完成一篇 100 章修仙超短篇小说的创作链路，验证可靠性、暴露体验痛点，并沉淀 token 成本、反 AI 味和知识库一致性分析结果。

覆盖流程

  • 世界观模板实例化：修仙模板、世界设定、角色卡、Skill 守护规则。
  • 碎片捕捉与 AI 整理：灵感碎片进入 PromptPipeline 并生成连贯段落。
  • 开篇引导：生成场景切入、人物切入、悬念切入三种开篇。
  • 章节管理：创建 100 章并验证章节级写作流程。
  • 连续生成：确定性适配器完成 100 章输出，验证 D-11 章节长度边界。
  • 故事结构：验证伏笔埋设、跨章跟踪、填坑解决和偏离检测。
  • 格式清洗与导出：验证 Markdown、TXT、JSON 三格式输出。
  • 分析报告：Token 成本、用户痛点、反 AI 味盲读、知识库一致性四类报告入口与详情页。

自动化验证结果

  • 通过：flutter analyze 无问题。
  • 通过：flutter test 全套测试通过，末尾结果为 All tests passed!。
  • 通过：确定性用户旅程测试完成 100 章流程。
  • 按预期跳过：缺少 GLM_API_KEY / OPENAI_API_KEY 时，真实外部模型测试跳过。

证据入口

  • test/journey/full_journey_test.dart — 端到端百章用户旅程。
  • test/journey/automated_ui_evidence_test.dart — 自动化 UI/旅程证据。
  • test/journey/helpers/story_outline.dart — 100 章修仙样例来源。
  • Phase 16 planning artifacts — 分析报告阶段计划、总结与验证文档。

已知限制与后续 UAT

  • 百章样例是确定性验证样例，主要证明链路闭环和效果参考，不代表最终文学质量。
  • 真实 GLM 长流程需要有效 API Key、稳定网络和更长运行时间。
  • Windows/Android 原生 IME、启动时间、生命周期恢复仍需物理设备人工验证。
  • 反 AI 味效果需要更广泛真实中文读者盲测，不能只依赖规则列表。
  • 模板文学性、节奏和人物弧线仍需人工编辑审读。
