Key Dart part-file rules (proven across 5 prior commits):
1. Main file: `library;` is FIRST line (before any imports). Then imports. Then `part 'x.dart';` directives.
2. Part files: FIRST line is `part of 'format_cleaner.dart';` — CANNOT contain import statements (they share the main file's imports).
3. `FormatChange` / `FormatChangeCategory` come from main file's `import '.../format_clean_result.dart';` → accessible in part files (shared import).
4. `RegExp` / `StringBuffer` are dart:core (auto-imported).
5. Move each section's private methods into `extension _Name on FormatCleaner { ... }`.
6. `clean()` calls moved methods via bare names (`_normalizeLineEndings`, etc.) — Dart resolves these through extension-on-this transparently. ZERO call-site changes.
7. The `// ----` section banners move with their methods into the part files.
