# Phase quick-260617-140 Plan 01: format_cleaner part-file split Summary

Mechanical part-file extraction of `format_cleaner.dart` (the largest hand-written file in the repo, 680 lines = 1.7× the 400 recommended max) into three cohesive part files, closing the repo-wide 800-line red-line campaign. Pure structural refactor — `FormatCleanResult` output is byte-identical for any input, proven by 1647 green tests.

## What Changed

`format_cleaner.dart` reduced from 680 → 71 lines. First line is now `library;` (before the import), followed by three `part` directives. The class body retains only `FormatCleanOptions` and the `FormatCleaner.clean()` shell — all 21 private section methods moved verbatim into three part files, each wrapped in a uniquely-named `extension _FormatCleaner<X> on FormatCleaner { }`.

`clean()` body is **unchanged** — bare method calls (`_normalizeLineEndings`, `_normalizePunctuation`, `_cleanMarkdownHeadings`, etc.) resolve transparently through Dart's extension-on-this mechanism. The orchestration order (whitespace → punctuation → markdown → paragraph) is load-bearing and untouched.
