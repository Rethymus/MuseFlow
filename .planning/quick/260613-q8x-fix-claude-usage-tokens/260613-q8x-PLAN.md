<objective>
Fix the Claude token-audit bug where `promptTokens` is always recorded as 0, drastically understating cost for Claude users.

Purpose: The Anthropic streaming API splits usage across two events — input (prompt) tokens arrive in `MessageStartEvent.message.usage.inputTokens` at stream start, output (completion) tokens arrive in `MessageDeltaEvent.usage.outputTokens` at stream end. The current `claude_adapter.dart` reads usage ONLY from `MessageDeltaEvent`, whose `inputTokens` field is null, so `promptTokens` is always 0. For writing tasks, input tokens (amplified by knowledge injection) are 10-100x output tokens, so the cost audit violates the README "成本透明" promise. This is a pure correctness bug with a known root cause.

Output: A corrected `claude_adapter.dart` that captures both events, plus a behavioral regression test that would have caught this bug. Follows project TDD mandate (Red → Green).
</objective>

anthropic_sdk_dart streaming events (sealed class MessageStreamEvent):

```dart
// Event carrying INPUT (prompt) tokens — arrives at stream START.
// This is the event the current adapter FAILS to read.
class MessageStartEvent extends MessageStreamEvent {
  final Message message;            // message.usage.inputTokens == prompt tokens (int, non-null)
  const MessageStartEvent({required this.message});
}

// Event carrying OUTPUT (completion) tokens — arrives at stream END (final cumulative).
// usage.inputTokens here is NULL (this is why the bug produces 0).
class MessageDeltaEvent extends MessageStreamEvent {
  final MessageDelta delta;
  final MessageDeltaUsage usage;    // usage.outputTokens (int, required), usage.inputTokens (int?, nullable)
  const MessageDeltaEvent({required this.delta, required this.usage});
}

// Text content delta — what the adapter extracts as String chunks.
class ContentBlockDeltaEvent extends MessageStreamEvent {
  final int index;
  final ContentBlockDelta delta;    // delta is TextDelta for text
  const ContentBlockDeltaEvent({required this.index, required this.delta});
}
class TextDelta extends ContentBlockDelta {
  final String text;
  const TextDelta(this.text);
}
```

anthropic_sdk_dart support types needed to build synthetic events:

```dart
class Message {
  final String id;
  final MessageRole role;           // defaults to MessageRole.assistant
  final List<ContentBlock> content; // pass [] for synthetic event
  final String model;
  final Usage usage;                // usage.inputTokens / usage.outputTokens (both int)
  const Message({required this.id, this.role = MessageRole.assistant,
                 required this.content, required this.model, required this.usage});
}

class Usage {                       // anthropic's OWN Usage (different from openai's Usage)
  final String type;                // pass 'message_start' for synthetic
  final int inputTokens;
  final int outputTokens;
  final int cacheCreationInputTokens; // default 0
  final int cacheReadInputTokens;     // default 0
  const Usage({required this.type, required this.inputTokens, required this.outputTokens,
               this.cacheCreationInputTokens = 0, this.cacheReadInputTokens = 0});
}

class MessageDelta {
  const MessageDelta();             // all fields optional/nullable — pass empty ctor
}

class MessageDeltaUsage {
  final int outputTokens;           // required
  final int? inputTokens;           // nullable — THIS IS THE BUG SOURCE (null in real streams)
  const MessageDeltaUsage({required this.outputTokens, this.inputTokens});
}
```

openai_dart Usage (what the adapter EMITS via onUsage):

```dart
class Usage {
  final int promptTokens;           // required — MUST be input tokens now
  final int? completionTokens;      // optional — output tokens
  final int totalTokens;            // required — prompt + completion
  const Usage({required this.promptTokens, this.completionTokens, required this.totalTokens});
}
```

Correct two-event pattern (confirmed in SDK example streaming_example.dart:124-127):

```dart
case MessageStartEvent(:final message): inputTokens = message.usage.inputTokens;
case MessageDeltaEvent(:final usage): outputTokens = usage.outputTokens;
```

Reference: OpenAIAdapter (lib/features/ai/infrastructure/openai_adapter.dart) uses ChatStreamAccumulator correctly — that adapter does NOT have this bug because openai_dart delivers usage in a single final chunk. This bug is Claude-specific due to Anthropic's two-event streaming design.
