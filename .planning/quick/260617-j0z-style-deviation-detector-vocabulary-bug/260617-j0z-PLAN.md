# quick-260617-j0z: style_deviation_detector vocabulary 维度双量尺门槛 bug

## 根因（orchestrator 已精确确认，活跃 bug）

`StyleDeviationDetector._analyzeVocabulary`（测量方，style_deviation_detector.dart:219）
与 `StyleAnalyzer._computeVocabularyRichness`（基线构建方，style_analyzer.dart:165）
**公式完全一致**——unique CJK 字符 type-token ratio 经 `((ratio - 0.25) / 0.30).clamp(0.0, 1.0)`
归一化。**唯一差异是最低字数门槛**：

| 方 | 门槛 | 角色 |
|----|------|------|
| StyleAnalyzer | `if (cjkChars.length < 50) return 0.5;` | profile 基线（权威） |
| StyleDeviationDetector | `if (cjkChars.length < 20) { ... textValue: 0.5 }` | AI 文本测量 |

**后果**：20-49 字 AI 文本，detector 用稀薄样本算出真实词汇丰富度分（小样本 type-token
方差大），去对比 analyzer 用 50+ 字稳健数据构建的 `profile.vocabularyRichness`——稀薄
测量 vs 稳健基线，vocabulary 维度偏差分失真（反AI味核心信号）。

**同类**：与 260617-hnl（rhythm 双量尺 `< 3` vs `< 5`）、260617-f7l（emotionalTone 双量尺）
完全同源——测量 ruler 与基线 ruler 的"何时可计算"门槛不一致。本任务是双量尺消除战役第 6 维
（vocabulary），与已闭合的 rhythm(hnl)/emotionalTone(f7l) 并列。

**审计旁注（同一调查，本次不改）**：`sentenceLength` 维度 detector `_analyzeSentenceLength`
仅 `lengths.isEmpty` 守卫、无数门槛，与 analyzer `_computeSentenceStats`（同样仅 isEmpty
守卫）一致——非活跃 bug，门槛一致。`_extractSentenceLengths` 两文件逻辑一致（仅
`_extractCjkChars` vs `_cjkCharCount` 方法名差异，hnl 计划已确认）。`_computeRhetoricHabits`
两文件此刻一致（hnl 计划已确认，属未来漂移风险，可选后续抽共享 StyleAnalysisUtils）。
故 5 维中仅 vocabulary 仍是活跃双量尺 bug。

## 修复（单行 + 注释）

`style_deviation_detector.dart:224` `cjkChars.length < 20` → `< 50`，对齐 analyzer 基线
门槛。加注释：测量方须与基线方用同一"何时可计算词汇丰富度"规则，避免稀薄样本测量对比
稳健基线（260617-hnl/f7l 同源原理：测量 ruler == 基线 ruler）。
