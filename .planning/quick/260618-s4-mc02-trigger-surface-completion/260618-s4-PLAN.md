# MC-02 Slice 4 — Complete the summary-trigger surface

## Objective
Slice 3 (260618-jn6) wired ONLY `ChapterNotifier.save` to fire-and-forget summary refresh. The other content-mutating/deleting ops are unwired, leaving two real gaps:

1. **Correctness bug (splitChapter)**: the original chapter's content is replaced with `beforeContent` (a SUBSET, shorter). Its stored summary still describes the WHOLE chapter, but `ChapterSummaryStalenessChecker.isStale` returns `false` when `currentWordCount <= sourceWordCount` (growth is negative) → the now-FACTUALLY-WRONG summary is marked fresh and injected on the next editor AI call. Force-refresh is required (bypass the staleness check, because the content was fundamentally replaced, not grown).
2. **Orphan hygiene (delete / mergeChapters chapter2)**: deleted/merged-away chapters leave orphan `ChapterSummary` rows in Hive; `ChapterSummaryRepository.getByManuscriptId` returns them. Clean up via `repository.delete(chapterId)`.

## Verified facts (do not re-derive)
- `ChapterRepository.add` **returns** a `Chapter` with a real uuid when input `id` is empty (`chapter_repository.dart:20-22`: `final id = chapter.id.isEmpty ? _uuid.v4() : chapter.id;`). Current `add`/`duplicateChapter`/`splitChapter`/`mergeChapters` DISCARD this return — the wiring MUST capture it to refresh on the correct id.
- `ChapterSummaryRefreshService` (slice 3, `chapter_summary_refresh_service.dart`) has: `refreshIfNeeded(chapter,{now})`, `refresh(chapter,{now})` (force), `_summarizeAndPut`, `minSummaryChars=20`. It does NOT yet have a delete method — **add `deleteSummary(chapterId)`**.
- `ChapterSummaryRepository.delete(String chapterId)` exists (`chapter_summary_repository.dart:67`), rethrows StateError.
- `_maybeRefreshSummary(Chapter chapter)` already exists in `chapter_notifier.dart:52` → calls `service.refreshIfNeeded`. Add a `force` param + a new `_deleteSummary` helper.
- `chapter_notifier_test.dart` uses `ProviderContainer` + repository overrides, does NOT override `chapterSummaryRefreshServiceProvider` → in existing tests the service is null (no AI provider configured) → helpers no-op. New wiring tests override `chapterSummaryRefreshServiceProvider` with a recording fake.
