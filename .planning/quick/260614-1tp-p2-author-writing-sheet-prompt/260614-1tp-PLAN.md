    **GREEN — 抽取器** `lib/features/editor/application/lexical_signature_extractor.dart`：
    - `class LexicalSignatureExtractor { const LexicalSignatureExtractor(); static LexicalSignature extract(String text, {int maxTerms = 15}) {...} }`
    - 算法步骤：
      1. **CJK 分片**：用 `RegExp(r'[一-鿿豈-]+')`（CJK Unified 0x4E00-0x9FFF + Extension A 0x3400-0x4DBF，可用 `[一-鿿㐀-䶿]+`）找出所有连续 CJK 片段，非 CJK（标点/空格/拉丁）作为分隔。每个片段是一个 CJK 字符串。
      2. **n-gram 生成**：对每个长度 ≥2 的片段，滑动窗口提取 bigram（步长1，窗口2）和 trigram（片段长度 ≥3 时，窗口3）。
      3. **频率统计**：用 `Map<String, int>` 统计每个 n-gram 的出现次数。
      4. **停用词过滤**：丢弃 `CjkStopwords.grams.contains(gram)` 的项。同时丢弃长度为1的项（bigram/trigram 本就 ≥2，但保险起见）。
      5. **显著度计算**：`score = frequency.toDouble() * weight`，其中 trigram weight=1.5、bigram weight=1.0。
      6. **排序**：按 score 降序。
      7. **去重（子串重叠）**：遍历排序后的候选，若某候选是已入选项的子串或超串且分数接近，保留高分者（实现简化：若候选 term 完全包含于已入选 term 或反之，跳过低分者）。注意不要过度去重把"拔剑"和"拔剑四"都砍掉——只去重完全重叠语义的。简化策略：若候选 term 是已入选 term 的子串（或反之），且长度差 ≤1，跳过。
      8. **截断**：取前 maxTerms，构造 `List<LexicalTerm>`，返回 `LexicalSignature(topTerms: ...)`。
    - 空文本/纯标点/纯拉丁：步骤1 分片结果为空 → 返回 `LexicalSignature.empty`。

    **REFACTOR**：若 extract 内联过长，抽取私有 helper（`_extractCjkSegments`/`_generateNGrams`/`_computeScore`），每个 < 30 行。不改变行为，重跑测试必须全绿。
