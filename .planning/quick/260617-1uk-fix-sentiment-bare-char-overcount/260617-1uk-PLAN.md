<objective>
Fix a substring-overcount bug in `style_deviation_detector.dart` where bare single-char `'爱'` (in the positive Set) and bare `'恨'` (in the negative Set) are matched as substrings by `String.allMatches`, producing inflated sentiment counts. The worst case is polarity inversion: `'恨不得'` (eager to, positive connotation) gets counted as a *negative* hit via the bare `'恨'`, dragging warmth/intensity in the wrong direction.

Purpose: The emotionalTone dimension feeds the AI-scent deviation score — a product-soul "反AI味" feature. Substring false-positives distort warmth/intensity and ultimately the deviation alarm. This surgical fix mirrors the recent bug-fix precedent (commit 9122b01) in the same file and aligns the inline lexicon with the curated shared `SentimentLexicon` design philosophy (which deliberately omits bare single-char entries).

Output: Patched detector (two single-line deletions) plus two regression tests pinning the corrected intensity measurement.
</objective>

```dart
int _countPositiveSentiment(String text) {
  const positives = <String>{
    // ... '温暖', '幸福', ... '欣慰', '爱',  // <-- REMOVE THIS LINE
    '喜欢', '珍惜',
  };
  var count = 0;
  for (final word in positives) {
    count += word.allMatches(text).length;   // allMatches does SUBSTRING matching
  }
  return count;
}

int _countNegativeSentiment(String text) {
  const negatives = <String>{
    // ... '难过', '害怕', '恨',  // <-- REMOVE THIS LINE
    '厌恶',
  };
  var count = 0;
  for (final word in negatives) {
    count += word.allMatches(text).length;
  }
  return count;
}
```

Why it breaks (load-bearing detail): `String.allMatches` returns every non-overlapping substring occurrence. A 1-char needle `'爱'` matches inside `可爱`/`爱好`/`爱情`/`爱不释手`/`亲爱`/`可爱极了`. A 1-char needle `'恨'` matches inside `恨不得` (positive sense!) / `悔恨`/`愤恨`/`可恨`. This double-counts and inverts polarity.

Why fix inline rather than import shared lexicon: the detector lives in `features/editor/application/`. `sentiment_lexicon.dart` lives in `features/editor/infrastructure/`. Per `.claude/rules/02-museflow-architecture.md`, application MUST NOT depend on infrastructure (reverse dependency forbidden). The inline Set is the correct architectural placement; we only remove the bad entries.

- AFTER: `'爱'` removed → positiveCount = 0 on this fragment. Correct.
