## 零、关键环境事实（影响测试策略）

1. **Web 构建成功** (`flutter build web --release` exit 0; build/web 44MB; main.dart.js 4.8MB)
2. **Web 应用结构正常启动**: main.dart 执行完成（trace 标记 + MaterialApp.title 生效），flt-glass-pane + flt-scene-host 挂载，无 JS 错误
3. **无头环境渲染阻塞根因**: 此 WSL2 沙箱的 headless Chromium **`webgl2 = false`**（webgl1 可用）。CanvasKit 强依赖 WebGL2 → 场景宿主建好但无法绘制 canvas → 白屏。**这是环境限制，非应用 bug**。用户真实浏览器（带 GPU + WebGL2）可正常渲染。
4. **本环境无 Chrome/Chromium 二进制，无 kimi-webbridge 守护进程** → 无法做 Web 实时视觉走查
5. **结论**: Web 端"研发"已完成（构建+启动正常）。实时 UI 走查需在真实浏览器（kimi-webbridge 或用户手动）进行；本环境改用 **测试套件 + Phase 13 自动化脚本（FakeAdapter 无需真实 API 跑全流程）+ 代码审计** 作为验证手段

## 二、代码审计发现的真实 bug（mock 测试全部漏掉）

> 这些是 Phase 25（真实 API 验证）必须抓的，但本次代码审计已提前发现，可直接修。

### BUG-1 🔴 Claude token 审计严重低估（promptTokens 恒为 0） ✅ FIXED (58632e7)
- **位置**: `lib/features/ai/infrastructure/claude_adapter.dart`
- **现象**: `onUsage` 仅从 `MessageDeltaEvent.usage` 取 token，但 Anthropic 流式 API 的 input tokens 在 `MessageStartEvent.message.usage.inputTokens`，output tokens 在 `MessageDeltaEvent.usage.outputTokens`。
- **影响**: Claude 用户的 Token 审计 `promptTokens=0`，成本审计严重失真，违背"成本透明"承诺。
- **修复**: 增加 `MessageStartEvent` 分支捕获 `inputTokens`，与 delta 的 `outputTokens` 合并（共享 `_buildUsage` 助手 + `processStreamEvents` 测试缝）。新增 3 个行为回归测试（synthetic 事件序列断言 promptTokens=120/completion=45/total=165）。基线 1505→1508。

### BUG-2 🟡 审计 outputText 记录原始文本（重评：非 bug，设计选择）
- **位置**: `synthesis_notifier.dart:265` 与 `editor_ai_notifier.dart:199`
- **原始判断**: onUsage 在 handleDone 触发，读到反 AI 味处理**前**的原始文本，或存在时序竞态。
- **代码核查结论（2026-06-13）**: `onUsage` 在流式 `await for` 期间被适配器调用（最终 usage 事件到达时），此时 `accumulatedText`/`progressText` 已是**完整原始模型输出**。Dart 单线程，onUsage 与 state 拷贝在同事件循环中确定性交错，**无真正竞态**。审计 outputText = 原始 API 输出，这是**可辩护的**（审计 = API I/O 记录，记录实际生成/计费的原始文本；反 AI 味是免费本地步骤，不计成本）。
- **决定**: 不做"修复"。保持记录原始输出。若产品要求审计展示最终入库文本，应作为独立需求（新增字段 rawOutputText vs finalText）而非改写现有语义。

### BUG-3 🔴 偏差检测隐藏二次 API 调用（成本翻倍） ✅ FIXED (dfe3198)
- **位置**: `editor_ai_notifier.dart` `unawaited(checkDeviations(...))` → `deviation_detection_service.dart` `createStream`
- **现象**: 每次编辑器 AI 操作完成后，无条件触发第二次完整 LLM 调用做 Skill 偏差检测，对用户完全透明（不可见、不可控）→ 编辑器 AI 操作实际消耗 2x token。
- **对应**: v1.5 路线图 CP-01 P0
- **修复**: 改为用户可选——新增 `autoDeviationCheckProvider`（持久化 bool，默认 false）+ `SettingsRepository.getAutoDeviationCheck/saveAutoDeviationCheck` + 设置页 SwitchListTile 开关。调用点 `if (ref.read(autoDeviationCheckProvider))` 门控。TDD：默认 0 次偏差调用 / 开启 1 次。基线 1508→1510。
