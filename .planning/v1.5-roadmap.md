# MuseFlow v1.5 改进路线图

## 里程碑目标

**名称:** 真实创作验证与体验打磨
**核心命题:** 从"功能完整"到"作者愿意天天用"——消除所有阻碍真实创作体验的摩擦点

**前一版本 v1.4 完成状态:**
- 24个阶段，76个计划，1468个测试全部通过
- Claude/Anthropic 多模型适配器已实现
- Web 端构建验证通过
- 作者风格指纹、反AI味深度优化、知识库智能注入、长篇上下文链、编辑器AI操作增强、写作进度仪表盘

## v1.5 研究背景

### 当前痛点分析（基于 v1.4 用户旅程验证）

| 痛点 | 严重度 | 描述 |
|------|--------|------|
| **真实 API 未验证** | 🔴 Critical | 所有AI功能仅通过 mock 测试验证，未用真实 Claude/GPT/DeepSeek API 端到端验证 |
| **反AI味词库无实证** | 🟡 High | 258个禁用词基于领域知识，未经真实中文文学作品校准 |
| **编辑器大文档性能** | 🟡 High | 300K+ 字文档滚动和选区性能未经真机验证 |
| **Windows IME 未测** | 🟡 High | 中文五笔/搜狗输入法在 super_editor 中的行为未经验证 |
| **首次使用引导** | 🟠 Medium | Onboarding 存在但与真实新用户创作流程脱节 |
| **错误恢复** | 🟠 Medium | 网络断连、API限流后用户需手动重试，缺少自动恢复 |
| **模板文学质量** | 🟠 Medium | 预设模板 prose 未经过专业文学审核 |

### 前沿研究方向（2025-2026 AI写作工具趋势）

基于 v1.4-proposal.md 的研究基础和最新趋势：

1. **RAG 增强长篇创作** — 将用户已有稿件作为检索库，AI 续写时自动参考前文风格和设定
2. **风格迁移评估** — 不仅检测AI味，还评估改写后是否保持了作者独特的叙事声音
3. **写作节奏感知** — 根据用户写作时间模式智能建议休息和创作时段
4. **多模态灵感输入** — 图片/语音转文字作为碎片捕捉输入源
5. **协作式大纲规划** — AI 辅助生成结构化大纲，用户选择/修改分支

## v1.5 阶段规划

### Phase 25: 真实API端到端验证（优先级：🔴 Critical）

**目标:** 用真实 API 验证三段式核心流程完整性

**任务:**
- [ ] 25.1 编写 E2E 测试套件（使用真实 API Key，CI 中 skip）
  - Capture → Synthesis → Editor 全链路
  - OpenAI / Claude / DeepSeek 三种 provider 分别验证
  - 验证 adapter routing 正确性
- [ ] 25.2 Claude 适配器真实流式验证
  - 验证 system prompt 提取
  - 验证多轮对话注入
  - 验证 usage 统计回调
- [ ] 25.3 错误恢复路径验证
  - API key 无效 → 中文错误提示
  - 网络断连 → 自动重试或优雅降级
  - 429 限流 → 等待重试提示
- [ ] 25.4 修复发现的真实 API 问题

**验收标准:** OpenAI + Claude 两种 provider 可完整走通碎片捕捉→AI整理→编辑器打磨流程

### Phase 26: 反AI味实证优化（优先级：🟡 High）

**目标:** 用真实中文文学作品校准禁用词库和检测算法

**任务:**
- [ ] 26.1 收集10篇经典中文小说片段（公版）作为测试集
- [ ] 26.2 运行反AI味检测，分析假阳性率
- [ ] 26.3 根据假阳性调整禁用词分类和权重
- [ ] 26.4 用 AI 生成文本验证假阴性率
- [ ] 26.5 优化 PromptPipeline 中的反AI味指令

**验收标准:** 禁用词库假阳性率 < 5%（公版文学作品不应被标记）

### Phase 27: Windows 真机验证与性能调优（优先级：🟡 High）

**目标:** 确保核心场景在物理 Windows 设备上流畅运行

**任务:**
- [ ] 27.1 Windows 桌面构建和安装测试
  - 安装包大小 < 100MB
  - 启动时间 < 3秒
  - 窗口管理（最小化/恢复/关闭）
- [ ] 27.2 中文 IME 测试
  - 搜狗输入法：拼音输入、候选词选择、词组输入
  - 微软五笔：编码输入、候选词、中英切换
  - super_editor 中的光标位置和 composing 状态
- [ ] 27.3 大文档性能测试
  - 300K 字文档滚动帧率 ≥ 55fps
  - 长文本选区响应时间 < 100ms
  - AI 操作后 diff 计算时间 < 500ms
- [ ] 27.4 性能瓶颈修复

**验收标准:** Windows 桌面三段式流程完整可用，IME 无异常

### Phase 28: 首次使用体验优化（优先级：🟠 Medium）

**目标:** 新用户 5 分钟内完成第一段创作

**任务:**
- [ ] 28.1 重设计 Onboarding 流程
  - 跳过式问卷（非强制）
  - 提供"快速开始"模板
  - AI Provider 配置引导（含 Claude 选项）
- [ ] 28.2 首次使用引导气泡
  - 碎片捕捉："试试记录你的第一个灵感"
  - AI 整理："选中碎片，点击合成"
  - 编辑器打磨："选中文字，查看AI建议"
- [ ] 28.3 预设 Provider 快速配置
  - Claude (Anthropic) 一键配置
  - DeepSeek 一键配置
  - Ollama 本地模型一键配置

**验收标准:** 新用户从安装到完成第一段AI辅助创作 < 5分钟

### Phase 29: 写作体验精细化（优先级：🟠 Medium）

**目标:** 消除日常写作中的摩擦点

**任务:**
- [ ] 29.1 AI 操作流式预览优化
  - 更平滑的打字机效果
  - 取消操作后保留已生成内容（部分保留）
  - 生成进度指示器
- [ ] 29.2 知识库注入可视化
  - 注入了哪些角色/设定 → 在 AI 响应中标注来源
  - 注入信心度 → 让用户知道 AI 参考了多少上下文
- [ ] 29.3 编辑器快捷操作增强
  - 常用 AI 操作一键快捷键绑定
  - 最近使用的操作快速重复
- [ ] 29.4 导出格式增强
  - DOCX 导出支持
  - EPUB 导出支持（基础）

**验收标准:** 用户调研显示"日常使用摩擦点减少 > 50%"
