## Phases

### Phase 11: 文稿库与章节管理

**Goal**: 将 MuseFlow 从单一编辑器升级为多文稿管理平台，支持文稿 CRUD、章节实体与导航、编辑器章节级切换、数据迁移和模板策略修订
**Depends on**: Phase 10 (v1.1 shipped — editor and knowledge base domain entities exist)
**Plans**: 6 plans

Plans:

- [x] 11-01: Domain entities (Manuscript, Chapter, Genre, ChapterExport) + Hive TypeAdapters + super_editor_markdown install (wave 1)
- [x] 11-02: Repositories, notifiers, auto-save service, purge service, sort utility, provider registration (wave 2, depends on 11-01)
- [x] 11-03: ManuscriptLibraryPage, ManuscriptCard, ManuscriptCreateDialog, ManuscriptSettingsPage, route wiring (wave 3, depends on 11-02)
- [x] 11-04: EditorWithSidebar, ChapterSidebar, chapter dialogs, document switching, auto-save integration (wave 4, depends on 11-02, 11-03)
- [x] 11-05: ExportBundle chapters, template chapter skeleton, AI chapter context, StatusBar progress, purge startup (wave 5, depends on 11-04)
- [x] 11-06: Gap closure — forced-save guarantees, chapter loading on editor entry (gap closure)

**Details:**

**Wave 1 (foundation):**
- Manuscript domain entity (immutable, copyWith, fromJson/toJson, equality)
- Chapter domain entity (immutable, copyWith, computed wordCount, fromJson/toJson)
- ManuscriptGenre utility (14 presets, WCAG AA color mapping, status constants)
- ChapterExport model (title, sortOrder, content for structured export)
- ManuscriptAdapter Hive TypeAdapter (typeId=2)
- ChapterAdapter Hive TypeAdapter (typeId=9)
- super_editor_markdown package installed

**Wave 2 (services):**
- ManuscriptRepository with CRUD + soft delete + purge queries
- ChapterRepository with CRUD + manuscript-scoped queries + document content update
- ManuscriptPurgeService with 30-day cascade deletion
- ManuscriptNotifier with create (auto-creates first chapter), save, softDelete, purgeDeleted, searchByTitle
- ChapterNotifier with loadChapters, add, save, delete, reorder (sequential sortOrder), duplicateChapter, splitChapter, mergeChapters
- ChapterAutoSave with 2s debounce + forceSave
- ManuscriptSortMode enum + compareManuscripts comparator
- All 6 providers registered in providers.dart

**Wave 3 (library UI + routing):**
- ManuscriptLibraryPage with genre-colored card grid
- ManuscriptCard with cover letter, genre color, word count
- ManuscriptCreateDialog with title/genre/cover letter inputs
- ManuscriptSettingsPage for manuscript metadata editing
- Route wiring in app.dart

**Wave 4 (editor + sidebar):**
- EditorWithSidebar wrapping editor with chapter navigation
- ChapterSidebar with drag-reorder, context menu (rename/split/merge/duplicate/delete)
- ChapterCreateDialog, ChapterRenameDialog, ChapterContextMenu
- Document switching via ValueKey strategy
- WidgetsBindingObserver for lifecycle save
- Keyboard shortcuts (Ctrl+S manual save)

**Wave 5 (integration):**
- Chapter-aware export (per-chapter content in Markdown/TXT)
- Template chapter skeleton (auto-create chapters from template)
- Chapter context middleware in AI prompt pipeline
- Startup purge of expired soft-deleted manuscripts

**Wave 6 (gap closure):**
- Load first persisted chapter on editor entry
- Forced save before opening manuscript settings
- Dispose no longer flushes (persistence via explicit awaited forceSave)
- Lifecycle best-effort async save with catchError
