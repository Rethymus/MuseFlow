<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User concept and edits -> AI completion prompt | User-provided creative text is sent to configured AI only when user triggers completion |
| AI model output -> Draft parser | Untrusted model output is parsed as structured JSON before entering draft state |
| Draft state -> Hive repositories | User-approved selected draft entities are persisted locally |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Tampering | AI completion output | mitigate | Parse strict JSON, ignore unknown keys, validate through existing domain constructors before save, and preserve original draft on invalid output. |
| T-07-06 | Information Disclosure | Story concept sent to AI | mitigate | Only send concept when user explicitly taps AI completion. Use active user-configured provider/API key. Do not log concept or model responses. |
| T-07-07 | Repudiation | Template-created entities | accept | Phase 7 does not require audit logs. Result summary confirms what was created immediately after save. |
| T-07-08 | Denial of Service | Long AI output or malformed JSON | mitigate | Treat invalid output as failure and preserve draft. Existing domain field limits catch excessive field sizes before persistence. |
| T-07-SC | Tampering | Third-party supply chain | mitigate | No new packages. Uses existing OpenAIAdapter, Riverpod, Hive repositories, and Material widgets. |

</threat_model>
