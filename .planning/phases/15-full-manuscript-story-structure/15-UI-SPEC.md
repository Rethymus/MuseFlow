### JOURNEY-08: Format Cleaning Validation

**What is validated:** Full 100-chapter format cleaning with automated assertions on output quality.

| Existing Component | Validation Target | Source File |
|--------------------|-------------------|-------------|
| `FormatCleaner` | Full cleanup pipeline: punctuation, markdown, whitespace, indentation, paragraph | `format_cleaner.dart` |
| `FormatCleanResult` | Result entity with changes list and categories | `format_clean_result.dart` |
| `FormatCleanPreviewDialog` | Preview diff display (manual spot-check only) | `format_clean_preview_dialog.dart` |
| `_FinishExportSection` | "Preview cleanup" button triggers dialog | `story_structure_page.dart` |

**Automated assertions (three categories from CONTEXT.md D-08):**

1. **Markdown residue cleanup** -- After cleaning, output must NOT match regex `/(^|\n)#{1,6}\s/`, `/\*\*[^*]+\*\*/`, `/```/`, `/`{1,3}[^`]+`{1,3}/`
2. **CJK punctuation normalization** -- After cleaning, output must NOT contain ASCII punctuation in CJK context: `/[一-鿿][,;:!?]/` should not match (mixed Chinese text with ASCII punctuation)
3. **Layout normalization** -- After cleaning, output must NOT contain 3+ consecutive blank lines: `/\n{4,}/`

**Additional assertions:**
- `FormatCleanResult.hasChanges` is true for raw AI-generated content (AI output always needs some cleanup)
- `FormatCleanResult.cleanedText` length <= `originalText` length (cleanup only removes/replaces, never adds significant content)
- Each chapter produces a valid `FormatCleanResult` with zero errors
- Running format cleaning twice on the same text produces no additional changes (idempotent)

**Manual spot-check (if desired):**
- Open `StoryStructurePage` -> "整理与导出" tab
- Tap "Preview cleanup" button
- Verify `FormatCleanPreviewDialog` shows change count and categorized changes
- Verify change diff shows original (red strikethrough) and replacement (green) for each change
- Apply cleanup, verify content is updated in editor

### JOURNEY-09: Three-Format Export Validation

**What is validated:** Full 100-chapter manuscript export to three formats with multi-layer assertions.

| Existing Component | Validation Target | Source File |
|--------------------|-------------------|-------------|
| `ExportService` | Three format export: TXT, Markdown, JSON | `export_service.dart` |
| `ExportBundle` | Bundle entity with chapters, foreshadowing, plot nodes, metadata | `export_bundle.dart` |
| `ExportDialog` | Format selection, path picking, export progress/success/error states | `export_dialog.dart` |
| `_FinishExportSection` | "Export manuscript" button triggers dialog | `story_structure_page.dart` |
| `ChapterExport` | Chapter export model with title and content | `chapter_export.dart` |

**Automated assertions (multi-layer from CONTEXT.md D-09):**

1. **Markdown format:**
   - Output contains 100 chapter title lines matching pattern `# 第\d+章` (or `## 第\d+章`)
   - Each chapter title line is followed by non-empty content
   - Chapter titles appear in sequential order
   - No Markdown syntax errors (unmatched `**`, `__`, etc.)

2. **TXT format:**
   - Output does NOT contain Markdown syntax characters: `#`, `**`, `~~`, `` ` ``, `[]()`
   - Output contains 100 chapter separators or title lines
   - File is pure plain text with no binary characters
   - Content is readable end-to-end

3. **JSON format:**
   - Output is valid JSON (parseable by `jsonDecode`)
   - Root object contains `chapters` array with length == 100
   - Each chapter object contains required metadata fields: `title`, `content`, `wordCount` (or equivalent), `createdAt` (or `timestamp`)
   - JSON includes foreshadowing entries, plot nodes, guardian annotations, character cards, world settings, skill documents
   - `exportedAt` timestamp is present and recent
   - `schemaVersion` field is present

4. **Cross-format assertions:**
   - All three formats contain the same chapter content (text body matches)
   - Total file size for 100 chapters is within 30,000-50,000 character range (CONTEXT.md D-09)
   - No chapter has empty content in any format

**Manual spot-check (if desired):**
- Open `StoryStructurePage` -> "整理与导出" tab
- Tap "Export manuscript" button
- In `ExportDialog`, verify format selector shows TXT/Markdown/JSON
- Select each format, verify description text changes
- Verify chapter count display shows "100 chapters"
- Select JSON format, verify summary panel lists all metadata categories with counts
- Enter path and export, verify success message
