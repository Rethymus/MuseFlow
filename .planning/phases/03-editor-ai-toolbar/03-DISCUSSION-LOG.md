## AI替换结果的呈现方式

| Option | Description | Selected |
|--------|-------------|----------|
| 直接替换（流式覆盖） | AI结果流式显示在原文位置，所见即所得。类似Google Docs的Smart Compose体验。 | |
| 侧边预览确认后替换 | AI结果先显示在侧边预览面板（类似Phase 2的合成面板），用户确认后才替换到编辑器。 | |
| 内联Diff逐处确认 | 在编辑器内高亮显示差异——删除的内容标红，新增的内容标绿。用户可以逐处接受或拒绝。 | ✓ |

**User's choice:** 内联Diff逐处确认
**Notes:** 用户选择了最精细的方案。小说创作场景下，逐句确认最安全——不怕误改好句子。

### Diff粒度

| Option | Description | Selected |
|--------|-------------|----------|
| 句子级 Diff | 每个句子独立高亮，逐句接受/拒绝。适合小说创作——改写通常以句子为单位。 | ✓ |
| 词级 Diff | 每个词独立高亮。太碎了，小说改写一般不会只改一个词。 | |
| 段落级 Diff | 整段替换，只能全部接受或全部拒绝。简单但可能混入不想改的内容。 | |

**User's choice:** 句子级 Diff

### 接受/拒绝交互

| Option | Description | Selected |
|--------|-------------|----------|
| 内联 ✓ ✕ 按钮 | 每个高亮句子旁边出现 ✓ ✕ 小按钮，点击逐个确认。 | |
| 选中时浮现操作栏 | 选中句子时才浮现操作栏（类似浏览器选文弹出），不选中时只显示颜色差异。 | ✓ |
| 全局操作栏 + 逐个 | 底部或顶部有一个全局操作栏：「全部接受」「全部拒绝」「逐个确认」。 | |

**User's choice:** 选中时浮现操作栏

### 未确认Diff处理

| Option | Description | Selected |
|--------|-------------|----------|
| 会话级（不持久化） | Diff状态只存在于当前会话——关闭文档后未确认的Diff自动接受或丢弃。 | |
| 持久化到文档 | 未确认的Diff随文档保存。下次打开文档时仍高亮显示。 | |
| 离开时提醒确认 | 切换页面时弹窗提醒「有 N 处未确认的AI修改」，用户选择处理后再离开。 | ✓ |

**User's choice:** 离开时提醒确认

## 浮窗菜单的交互细节

### 菜单布局

| Option | Description | Selected |
|--------|-------------|----------|
| 横排按钮 | 三个按钮横排一行：「语气改写」「文段润色」「自由输入」。紧凑，类似浏览器选文弹出。 | ✓ |
| 图标卡片式 | 用图标+文字的卡片式布局，每个操作有简短说明。更丰富但占用更多空间。 | |
| 展开式子菜单 | 一个主按钮展开子菜单，类似右键菜单。最省空间但多一次点击。 | |

**User's choice:** 横排按钮

### 自由输入入口

| Option | Description | Selected |
|--------|-------------|----------|
| 浮窗内嵌输入框 | 点击后在浮窗下方展开一个小型文本输入框。用户输入指令，回车触发AI。 | ✓ |
| 独立弹窗 | 点击后弹出一个独立对话框，有更大的输入空间。 | |
| 浮窗就地转换 | 在浮窗菜单中直接加一个快捷文本输入，点击就变成输入框。 | |

**User's choice:** 浮窗内嵌输入框

### 处理状态

| Option | Description | Selected |
|--------|-------------|----------|
| 浮窗内进度条 | AI处理时，浮窗变成进度条+取消按钮。不遮挡编辑器其他区域。 | ✓ |
| 选区上方文字提示 | AI处理时在选区上方显示「正在改写...」文字提示，带动画。 | |
| 选区变灰+浮窗loading | AI处理时整个选区变灰/半透明，浮窗显示loading状态。 | |

**User's choice:** 浮窗内进度条

### 菜单位置

| Option | Description | Selected |
|--------|-------------|----------|
| 紧贴选区（智能翻转） | 浮窗紧跟选区下方。选区在屏幕顶部时浮窗出现在下方，选区在底部时浮窗出现在上方。 | ✓ |
| 固定在编辑器顶部 | 浮窗固定在编辑器顶部（替换原有格式工具栏的位置）。 | |
| 固定在底部 | 浮窗固定在屏幕底部，类似移动端的BottomSheet。 | |

**User's choice:** 紧贴选区（智能翻转）

## 文本来源追踪的视觉方案

### 标记方式

| Option | Description | Selected |
|--------|-------------|----------|
| Attribution底色标记 | 用super_editor的Attribution系统给AI修改的文字加底色（如淡蓝色背景）。 | ✓ |
| 段落侧边指示器 | 在段落左侧加竖线或圆点标记「这段有AI修改」。 | |
| 字体样式区分 | 用不同的字体样式区分（如AI修改用楷体，人工用宋体）。 | |

**User's choice:** Attribution底色标记

### 标记移除时机

| Option | Description | Selected |
|--------|-------------|----------|
| 接受后自动移除 | 用户接受Diff后标记自动移除。最终文档中不含任何AI标记。 | ✓ |
| 保留到手动清除 | 标记一直保留到用户手动清除。 | |
| 接受后移除+保留元数据 | 接受后标记移除，但保留不可见的元数据记录。 | |

**User's choice:** 接受后自动移除

### 标记样式

| Option | Description | Selected |
|--------|-------------|----------|
| 淡蓝底色 | 淡蓝色半透明底色（如Colors.blue.withValues(alpha: 0.1)）。在深色主题下不刺眼。 | ✓ |
| 虚线下划线 | 用虚线下划线标记。不占背景空间，更轻量。 | |
| 波浪下划线 | 用波浪线下划线（类似拼写检查的红线）。 | |

**User's choice:** 淡蓝底色

### 修改统计

| Option | Description | Selected |
|--------|-------------|----------|
| 显示待确认数量 | 在编辑器状态栏或浮窗中显示「当前文档有 N 处AI修改待确认」。 | ✓ |
| 不显示统计 | 不显示额外统计。用户通过颜色高亮自行判断。 | |
| 详细修改清单 | 在文档开头或底部显示详细的修改清单。 | |

**User's choice:** 显示待确认数量

## 上下文锚点的选择交互

### 锚点模式

| Option | Description | Selected |
|--------|-------------|----------|
| 持久锚点 | 用户选中一段文字，点击「设为上下文锚点」。锚点标记一直保留，后续所有AI操作都自动参考。 | |
| 一次性参考 | 用户选中一段文字，标记为「本次参考」。AI操作完成后锚点自动清除。 | |
| 混合模式 | 支持两种模式——右键菜单可选择「设为持久锚点」或「本次参考」。 | ✓ |

**User's choice:** 混合模式

### 锚点入口

| Option | Description | Selected |
|--------|-------------|----------|
| 浮窗菜单集成 | 选中文字后，浮窗菜单中增加一个「📌 设为上下文」按钮。点击后弹出选择。 | ✓ |
| 右键菜单 | 右键选中的文字，上下文菜单中增加「设为上下文锚点」选项。 | |
| 快捷键 | 快捷键（如Ctrl+Shift+A）直接标记选中文字为锚点。 | |

**User's choice:** 浮窗菜单集成

### 锚点视觉

| Option | Description | Selected |
|--------|-------------|----------|
| 底色+图标 | 锚点文字加淡金色底色，段落左侧显示📌图标。持久锚点和一次性锚点用不同底色强度区分。 | ✓ |
| 仅侧边标记 | 锚点段落左侧显示一个小型锚点标记（如⚓），不改变文字底色。 | |
| 状态栏文字 | 在编辑器底部状态栏显示「上下文锚点：N 个持久，M 个临时」，不直接在文本中标记。 | |

**User's choice:** 底色+图标

### 锚点注入方式

| Option | Description | Selected |
|--------|-------------|----------|
| 自动注入系统消息 | 锚点内容自动注入到PromptPipeline的系统消息中，作为「参考上下文」section。 | ✓ |
| 用户手动选择注入 | 用户在浮窗菜单中可以勾选哪些锚点本次使用。 | |
| 注入用户消息 | 锚点内容注入到用户消息中，紧跟在选中文字之后。 | |

**User's choice:** 自动注入系统消息
