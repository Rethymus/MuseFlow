## Design System

| Property | Value |
|----------|-------|
| Tool | none (Flutter Material 3) |

## Typography

Inherited from `app_theme.dart` comment and `GoogleFonts.notoSansScTextTheme()`. Typography is constrained to 4 font sizes and 2 font weights: w400 regular and w600 semibold (w700 display matches existing convention from Phase 08).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Label | 12px | w500 (medium) | 1.3 | Chapter word counts, status badges, sort labels, timestamp labels, progress percentage |
| Body | 14px | w400 (regular) | 1.5 | Chapter titles, manuscript descriptions, card body text, form helper text, empty state descriptions |
| Heading | 20px | w600 (semibold) | 1.2 | Manuscript card titles, empty state headings, dialog titles, sidebar manuscript title |
| Display | 28px | w700 (bold) | 1.2 | AppBar page title "文稿库" (inherited from existing pattern) |

**Phase-specific typography:**
- Library card cover letter: 36px w700, centered on colored background, max 2 chars (per D-07)
- Chapter sidebar row title: 14px w400, single line, ellipsis overflow
- Chapter sidebar word count: 12px w500, right-aligned, `colorScheme.onSurfaceVariant`
- Status badge text: 12px w500, pill shape (reuse `_PassiveTag` pattern from template_gallery_page.dart)
- Progress bar label: 12px w500, "2,400 / 50,000" format

## Interaction Contracts

### Library Navigation

| Interaction | Behavior |
|-------------|----------|
| Tap manuscript card | Navigate to `/manuscript/:id/editor`, bottom nav hidden, show `EditorWithSidebar` with back button |
| Long-press/right-click manuscript card | Show context menu: "编辑信息" (Edit Info), "导出" (Export), "删除" (Delete) |
| FAB "创建文稿" button | Show `ManuscriptCreateDialog` (quick create) or navigate to detailed create page |
| Sort dropdown | Toggle between "最近编辑" (default), "创建时间", "标题" |
| Back button from editor | Forced save current chapter → navigate to library |

**Source:** CONTEXT.md D-11: "Library -> tap manuscript card -> enter Editor (full screen, bottom nav hidden) -> AppBar back button returns to Library."

### Chapter Sidebar Interactions

| Interaction | Behavior |
|-------------|----------|
| Tap chapter row | Save current chapter document (forced) -> load selected chapter document -> highlight row |
| Drag chapter row (reorder) | Visual reorder preview, recalculate `sortOrder` on drop |
| Long-press/right-click chapter row | Show context menu with: Rename, Split (at cursor), Merge with next, Duplicate, Delete |
| "新建章节" button | Open `ChapterCreateDialog`, create chapter after current chapter with `sortOrder = current + 1`, shift subsequent chapters |
| Inline rename | Show `ChapterRenameDialog` with pre-filled title |
| Keyboard: Ctrl+Up | Navigate to previous chapter |
| Keyboard: Ctrl+Down | Navigate to next chapter |
| Keyboard: Ctrl+Shift+N | Create new chapter |

**Source:** CONTEXT.md D-13: "Left sidebar panel is always visible within the editor (not collapsible). Fixed width ~240-280px." D-15: "Chapter reordering via drag & drop in sidebar." D-16: Chapter operations list.

### Chapter Operations

| Operation | Trigger | Behavior |
|-----------|---------|----------|
| Create | "新建章节" button or Ctrl+Shift+N | Dialog with title input, creates chapter after current, increments subsequent sortOrder |
| Rename | Context menu "重命名" | Dialog with TextField pre-filled, saves immediately on confirm |
| Reorder | Drag & drop in sidebar | Recalculate sortOrder values after drop |
| Split | Context menu "在光标处拆分" | Split current chapter at cursor position into two chapters, preserve content before/after cursor |
| Merge | Context menu "与下一章合并" | Merge current chapter with next adjacent chapter, append content, delete next chapter |
| Duplicate | Context menu "复制章节" | Clone chapter content as new chapter immediately after, with title suffix "(副本)" |
| Delete | Context menu "删除" | Confirmation dialog -> hard delete -> recalculate adjacent sortOrder |

**Source:** CONTEXT.md D-16: "Chapter operations: create, delete, rename, reorder, split, merge, duplicate."

### Auto-Save Behavior

| Trigger | Action |
|---------|--------|
| Edit (debounced) | Save chapter Document JSON 2-3 seconds after last keystroke |
| Chapter switch | Forced save of current chapter before loading new chapter |
| Back to library | Forced save of current chapter |
| App lifecycle pause | Forced save of current chapter |

**Source:** CONTEXT.md D-19: "Chapter content auto-saves with dual guarantee: Debounced save (2-3 seconds) + Forced save on chapter switch, back-to-library navigation, and app lifecycle pause."

### Manuscript Creation Flows

| Flow | Steps | Result |
|------|-------|--------|
| Quick create | Dialog: title + genre dropdown -> "创建" | Manuscript with 1 empty chapter |
| Detailed create | Page: title, genre, description, target word count, WorldSetting link, CharacterCard links -> "创建" | Manuscript with 1 empty chapter + linked entities |
| From template | Template selection -> concept input -> "创建" | Manuscript with preset chapter skeleton + WorldSetting + CharacterCards (per D-20) |

## State Preservation

| Concern | Solution |
|---------|----------|
| Chapter document switching | Each Chapter stores its own `documentJson` (SuperEditor serialized). Switching chapters: forced save current -> load new Document from JSON -> set on Editor. |
| Undo history per chapter | Each chapter's Document has independent undo stack (SuperEditor default). Switching chapters loads a fresh Document. |
| Auto-save on navigation | Forced save on chapter switch, back-to-library, and app lifecycle pause. Debounced 2-3s save during editing. |
| Sidebar scroll position | `ScrollController` preserved by `AutomaticKeepAliveClientMixin` on ChapterSidebar |
| Library scroll position | StatefulShellRoute.indexedStack preserves branch state |
| Sort preference | Persist sort mode in Hive `settings` box as `library_sort_mode` key |
