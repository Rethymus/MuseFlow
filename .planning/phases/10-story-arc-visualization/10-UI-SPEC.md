## Spacing Scale

Declared values (multiples of 4, matching existing codebase and Phase 08 UI-SPEC):

| Token | Value | Usage | Codebase Evidence |
|-------|-------|-------|-------------------|
| xs | 4px | Icon gaps, inline padding, tag vertical padding | Existing spacing pattern from Phase 08 |
| sm | 8px | Compact element spacing, toolbar padding, button gaps | `EdgeInsets.symmetric(horizontal: 16, vertical: 8)` in capture_page.dart |
| md | 16px | Default element spacing, page horizontal padding, card padding | `EdgeInsets.all(16)` in provider_card.dart, PlotNodeForm content spacing |
| lg | 24px | Section padding, icon sizes | `Icon(icon)` at default 24px; `SizedBox(height: 24)` in onboarding |

- 12px for form field vertical spacing (`const SizedBox(height: 12)` in PlotNodeForm) -- standard for form layouts

## Interaction Contracts

### Graph Navigation (Zoom/Pan)

| Interaction | Behavior |
|-------------|----------|
| Pinch-to-zoom | InteractiveViewer handles zoom (min 0.5x, max 3x) |
| Pan (single finger drag on canvas) | InteractiveViewer pans the viewport |
| Mouse wheel (desktop) | Zoom in/out at pointer position |

**Source:** RESEARCH.md: "graphview's GraphView.builder wraps everything in an InteractiveViewer.builder for free zoom/pan."

### Node Editing (Tap/Bottom Sheet)

| Interaction | Behavior |
|-------------|----------|
| Tap node | Open `NodeEditBottomSheet` via `showModalBottomSheet(isScrollControlled: true)` |
| Bottom sheet swipe down | Close bottom sheet, discard unsaved changes |
| "保存" button | Validate fields → call `PlotNodeNotifier.save()` → close sheet |
| "取消" button | Close bottom sheet, discard changes |
| Form field change | Update local state, no auto-save |

**Source:** CONTEXT.md D-13: "点击节点弹出底部表单（BottomSheet）进行编辑。符合Flutter移动端习惯。"

### Node Dragging (Position Persistence)

| Interaction | Behavior |
|-------------|----------|
| Long press + drag node | Update `node.position` in memory, set `algorithm.setFocusedNode(node)` |
| Drag end | Debounced save to `NodePositionRepository` after 1 second of inactivity |
| Position restoration | On graph load, read saved positions from `graph_positions` box, override algorithm positions |

**Source:** CONTEXT.md D-15: "节点位置使用独立映射表存储（NodePosition: PlotNode.id → Position）。PlotNode保持纯净，位置信息分离。" RESEARCH.md pattern: "Debounced save to repository -- _positionSaveTimer?.cancel(); _positionSaveTimer = Timer(const Duration(seconds: 1), ...)."

### Tab Switching

| Interaction | Behavior |
|-------------|----------|
| Tap "弧线图" tab | Switch to TabBarView index 4, show `StoryArcGraphPage` |
| Swipe left/right | Switch between tabs (Material TabController default behavior) |

### Empty State Interaction

| Interaction | Behavior |
|-------------|----------|
| Tap "创建第一个节点" button | Open `NodeEditBottomSheet` in create mode (node parameter is null) |

## State Preservation

| Concern | Solution |
|---------|----------|
| Graph rebuild losing node positions | NodePositionRepository persists positions to Hive `graph_positions` box, restored on load |
| PlotNode data changes | Watch `plotNodeNotifierProvider`, rebuild Graph when AsyncValue emits new data |
| Node drag position persistence | Debounced 1-second save to NodePositionRepository after drag ends |
| Tab switch disposal | `AutomaticKeepAliveClientMixin` on StoryArcGraphPage to preserve graph state across tab switches |
| Empty state vs graph state | Derived from `AsyncValue<List<PlotNode>>` -- empty list shows empty state, non-empty shows graph |

## Accessibility

| Concern | Solution |
|---------|----------|
| Graph nodes (tap targets) | Minimum 44px × 44px touch target (node size 120×48 or 160×64, surrounded by transparent padding) |
