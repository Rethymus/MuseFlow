    IMPORTANT -- Anti-patterns from RESEARCH.md:
    - Gesture conflict: Use the `_isDraggingNode` flag pattern. When dragging, InteractiveViewer should not capture the gesture. The graphview library's recommended pattern handles this via setFocusedNode + direct position mutation within the node's GestureDetector.
    - Synchronous Hive writes: MUST use debounced Timer (1 second), NOT direct save on every drag frame (Pitfall: performance)
    - Node identity: Reuse the same Node.Id objects across rebuilds (Pitfall 2). The nodeMap cache should be maintained outside the build method or cleared only when PlotNode list changes.
    - Algorithm shuffle: NEVER set shuffleNodes: true when saved positions exist (Pitfall 3)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-07 | Denial of Service | Excessive drag events flooding Hive | mitigate | Debounced save (1 second after last drag) per RESEARCH anti-pattern. Timer.cancel on each new drag event prevents queue buildup. |
| T-10-08 | Denial of Service | Corrupted positions causing NaN/Infinity coordinates | mitigate | NodePosition.fromDart coerces via `as num).toDouble()` which handles numeric inputs. Graph layout falls back to algorithm on position read failure per Plan 01 repository try/catch. |
| T-10-09 | Tampering | Position data corruption in Hive box | accept | Positions are non-sensitive (just x,y coordinates). Worst case: nodes jump to unexpected positions, user can re-drag. Repository try/catch throws StateError, graph falls back to algorithm layout. |
