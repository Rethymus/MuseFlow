```dart
class ContextAnchorMiddleware extends PromptMiddleware {
  const ContextAnchorMiddleware();
  @override
  PromptContext apply(PromptContext context) {
    final anchorRefs = context.anchors;
    if (anchorRefs == null || anchorRefs.isEmpty) return context;
    final buffer = StringBuffer();
    buffer.write('以下是作者指定的参考上下文，请在改写时参考：');
    for (final anchor in anchors) {
      buffer.write('\n\n【${anchor.label}】\n${anchor.text}');
    }
    return context.addMessage(ChatMessage.system(buffer.toString()));
  }
}
```

  <behavior>
    - KnowledgeInjectionMiddleware extends PromptMiddleware with const constructor
    - Collects scan text from PromptContext: selectedText + fragment texts + anchor texts
    - Calls NameIndex.findMatches on collected text
    - Deduplicates matches by entity ID, sorts by match count (most mentioned first)
    - Limits to top 5 entities within 30% token budget
    - Builds injection text: header + per-entity 【displayName】 + toContextString()
    - Appends to existing system message or creates new one
    - Returns context unchanged if no matches found
  </behavior>
  <action>
    lib/features/knowledge/application/knowledge_injection_middleware.dart:
    - Class `KnowledgeInjectionMiddleware extends PromptMiddleware`. Constructor takes `NameIndex`, `CharacterCardRepository`, `WorldSettingRepository`, `TokenBudgetCalculator`. Use `const` constructor where possible (note: repositories are not const, so the constructor cannot be const -- follow the same pattern as other non-const middlewares).
    - Method `_collectScanText(PromptContext context)`: builds a single string from `context.selectedText` (if non-null), all `context.fragments` text, and all `context.anchors` text (cast to AnchorReference, read .text).
    - In `apply()`:
      1. Collect scan text. If empty, return context unchanged.
      2. Call `_nameIndex.findMatches(scanText)`. If empty, return unchanged.
      3. Deduplicate: count matches per entityId. Sort by count descending.
      4. Calculate knowledge budget: `(context.tokenBudget * 0.3).toInt()` tokens.
      5. Iterate sorted entity IDs. For each: look up entity via `_charRepo.getById(id)` or `_settingRepo.getById(id)` (check `_nameIndex` entity type to decide which repo). If found, format as `\n\n【${entity.displayName}】\n${entity.toContextString()}`. Estimate tokens via `_budgetCalc.estimateTokens(entityText)`. If adding this entity would exceed budget, stop.
      6. If any entities were added, prepend header: `\n\n以下是与当前内容相关的角色和设定信息，请在创作时参考：` followed by all entity texts.
      7. If context has a system message (messages[0] is system), append to it via `replaceSystemMessage`. Otherwise, `addMessage(ChatMessage.system(...))`.
