  <action>
    lib/features/knowledge/presentation/quick_insert_dialog.dart:
    - `class QuickInsertDialog extends ConsumerStatefulWidget`.
    - State manages: `TextEditingController _searchController`, `String _query`, `EntityType? _filterType` (null = all, character/setting/skill).
    - Layout: Dialog with search TextField at top, filter chips below (全部, 角色, 世界观, 模板), scrollable list of matching entities below.
    - Search: as user types, filter entities whose `allNames` contains the query substring (case-insensitive). Use the repository providers to load all entities.
    - Filter chips: "全部" (no filter), "角色" (characters only), "世界观" (settings only), "模板" (skills only).
    - List items: ListTile showing entity displayName + type badge (角色/世界观/模板) + a short preview from toContextString (first 50 chars).
    - On tap: insert reference marker into editor and close dialog.
    - Reference marker format: `{{角色:${entity.name}}}` for characters, `{{世界观:${entity.name}}}` for settings, `{{模板:${entity.name}}}` for skills.
    - Insertion: access editor via `ref.read(editorProvider)`, use `_editor.execute([InsertTextRequest(text: marker, position: ...)])` at the current cursor position. If no selection, insert at caret. If selection, replace selection.
    - Keyboard: Escape closes dialog. Enter on first result selects it.
    - Empty state: "没有找到匹配的知识库条目" when search has no results.
    - Loading state: show CircularProgressIndicator while repositories load.

    Helper function `_buildEntityList`:
    - Combines entities from all three repositories (characterCardRepositoryProvider, worldSettingRepositoryProvider, skillRepositoryProvider).
    - Each entry is a tuple of (KnowledgeEntity, EntityType).
    - Filtered by _filterType and _query.
    - Sorted: exact name match first, then by name length (shorter first for relevance).
  </action>
