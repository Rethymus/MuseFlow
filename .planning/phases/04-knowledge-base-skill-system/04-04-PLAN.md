  <action>
    lib/features/knowledge/application/skill_enforcement_middleware.dart:
    - `class SkillEnforcementMiddleware extends PromptMiddleware`.
    - Constructor takes `SkillRepository` and `TokenBudgetCalculator`.
    - In `apply()`:
      1. Call `_skillRepo.getActive()`. If empty, return context unchanged.
      2. Build injection buffer: header `以下是当前激活的世界观设定约束，请严格遵守：`.
      3. For each active skill, append `\n\n【${skill.name}】\n` + relevant sections (rules, taboos, terminology -- the constraint-relevant sections, not all sections). Use `skill.sections.rules`, `skill.sections.taboos`, `skill.sections.terminology`. Skip null sections.
      4. Check token budget (20% of context.tokenBudget). If over budget, truncate from the last skill or reduce to fewer skills.
      5. Append to existing system message or create new one via `context.addMessage(ChatMessage.system(...))`.

    lib/features/knowledge/application/deviation_detection_service.dart:
    - `class DeviationDetectionService`.
    - Constructor takes `OpenAIAdapter` (or reads from provider).
    - `enum DeviationSeverity { low, medium, clear }`.
    - `class DeviationWarning` with fields: `String description`, `DeviationSeverity severity`, `String skillName`, `String? suggestedFix`. Include `fromJson` for parsing AI output.
    - `class DeviationResult` with field: `List<DeviationWarning> warnings`. Helper: `bool get hasWarnings => warnings.isNotEmpty`.
    - Method `Future<DeviationResult> detectDeviations(String text, List<SkillDocument> activeSkills)`:
      1. Build prompt: system instruction asking the AI to check the given text against the skill rules. Include the text and each skill's rules/taboo sections. Request JSON array output with fields: description, severity (low/medium/clear), skillName, suggestedFix.
      2. Call `openaiAdapter.createChatCompletionStream()` with maxTokens = 1024 (short output).
      3. Collect full response, parse JSON.
      4. Filter: only keep warnings with severity >= medium (per Pitfall 5 -- avoid noise).
      5. Return DeviationResult.
    - Handle parse failure gracefully: return empty result (don't crash on bad AI output).

    lib/core/presentation/providers.dart:
    - Add `skillEnforcementMiddlewareProvider` (FutureProvider, depends on skillRepositoryProvider and tokenBudgetCalculatorProvider).
    - Add `deviationDetectionServiceProvider` (Provider, depends on openaiAdapterProvider).

    - skill_enforcement_middleware_test.dart: mock SkillRepository returning active skills, verify system message contains skill rules/taboo/terminology. Test empty active skills -> no injection. Test budget exceeded -> truncation.
    - deviation_detection_service_test.dart: mock OpenAIAdapter, test prompt construction, test JSON parsing of deviation warnings, test filter removes low-severity, test graceful handling of unparseable AI output.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Active skill rules -> AI prompt | Skill constraints injected into system message sent to external AI API |
| AI deviation analysis -> user-facing warnings | AI-generated contradiction analysis shown to user |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Tampering | DeviationDetectionService output | mitigate | Warnings are advisory only (not blocking). User can dismiss any warning. AI output is never trusted as authoritative -- it's a suggestion. |
| T-04-08 | Information Disclosure | Skill rules in AI prompt | accept | Skill content is user's own creative world-building, not sensitive data. User controls what skills are active. |
</threat_model>
