    lib/features/knowledge/application/skill_generation_service.dart:
    - `class SkillGenerationService`.
    - Constructor takes `OpenAIAdapter`, `String apiKey`, `String baseUrl` (or reads from providers).
    - Method `Stream<String> generateSkillStream(String conceptDescription)`:
      1. Build system prompt: instruct AI to act as a world-building expert. Request structured output with markdown headings: `## еҠӣйҮҸзӯүзә§дҪ“зі»`, `## й—Ёжҙҫ/еҠҝеҠӣе…ізі»`, `## дё–з•Ңи§„еҲҷ`, `## зҰҒеҝҢ/йҷҗеҲ¶`, `## дё“з”ЁжңҜиҜӯ`. Ask for Chinese output. Include the user's concept description as the creative brief.
      2. Call `openaiAdapter.createChatCompletionStream()` with the prompt.
      3. Yield streaming chunks for real-time display.
    - Method `SkillDocument parseSkillDocument(String name, String description, String rawContent)`:
      1. Try JSON parse first (if AI returned JSON).
      2. Fall back to markdown heading-based parsing: split by `## ` headers, map to SkillSections fields.
      3. If neither works, store everything as `rawContent`.
      4. Return SkillDocument with id from Uuid, isActive = false, createdAt = now.

    lib/features/knowledge/application/skill_notifier.dart:
    - `class SkillGenerationNotifier extends AutoDisposeAsyncNotifier<SkillDocument?>`.
    - State: null when idle, SkillDocument when complete.
    - Additional field: `String progressText` (exposed via a separate provider or as part of state).
    - Method `Future<void> generateSkill(String conceptDescription, String skillName)`:
      1. Set state to loading.
      2. Call `skillGenerationService.generateSkillStream()`.
      3. Accumulate streaming text into progressText.
      4. When stream completes, parse into SkillDocument via `parseSkillDocument()`.
      5. Save to SkillRepository.
      6. Set state to the saved SkillDocument.
    - Method `void cancel()`: cancels ongoing stream.
    - Method `Future<void> saveEdited(SkillDocument doc)`: saves user-edited version to repository.

    - `class SkillListNotifier extends AsyncNotifier<List<SkillDocument>>`:
      - `build()` loads all from SkillRepository.
      - `Future<void> toggleActive(String id)` toggles isActive and refreshes.
      - `Future<void> delete(String id)` deletes and refreshes.
