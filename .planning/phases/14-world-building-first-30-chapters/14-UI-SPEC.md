# Phase 14 -- UI Design Contract

> Validation contract for the World-Building & First 30 Chapters phase.
> This phase runs automated journey scripts and manual spot-checks against
> existing UI. No new user-facing UI is built. The contract defines what
> existing interactions must be validated and what test fixtures must contain.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Flutter/Material 3, not React) |
| Preset | not applicable |
| Component library | Flutter Material 3 |
| Icon library | Material Icons (Icons.*) |
| Font | Noto Sans SC via google_fonts |

**Notes:** This is a Flutter/Dart project. shadcn gate does not apply. No new visual design in this phase. All UI validated by journey scripts and manual spot-checks was built in Phases 0-11 and follows the existing Material 3 dark indigo theme defined in `lib/shared/theme/app_theme.dart`.

---

## Phase Scope: No New UI

Phase 14 is a **user journey validation phase**. All required features shipped in v1.0--v1.2. The deliverables:

| Deliverable | Type | UI Impact |
|-------------|------|-----------|
| Journey container factory | Dart test helper (ProviderContainer + real GLM API) | None -- no widgets rendered |
| Xianxia fixtures | Dart data classes (character cards, world settings, Skill documents, story outline) | None -- test data only |
| 6 journey test files | Dart tests using ProviderContainer | None -- no widgets rendered |
| Structured issue log | Markdown document | None -- documentation only |
| Manual spot-check checklist | Markdown document | None -- documentation only |

**Design tokens (spacing, typography, color) are unchanged from Phase 11/13 UI-SPEC.** This phase validates that existing UI works correctly at scale (30 chapters, real AI generation), not that new UI looks correct.

## Structured Issue Log Contract

Problems discovered during journey validation are recorded in a structured format.

### File Location

`.planning/phases/14-world-building-first-30-chapters/14-ISSUE-LOG.md`

### Format

| Field | Values | Example |
|-------|--------|---------|
| Category | "功能缺陷" / "体验摩擦" / "缺失需求" | "功能缺陷" |
| Severity | "高" / "中" / "低" | "高" |
| Title | Short descriptive title | "30章创建后侧边栏滚动卡顿" |
| Reproduce steps | Numbered steps | "1. Create 30 chapters 2. Scroll sidebar rapidly 3. Observe jank" |
| Expected behavior | What should happen | "Smooth scrolling at 60fps" |
| Actual behavior | What actually happens | "Visible frame drops, ~20fps" |
| Evidence | Screenshot/log path | "logs/chapter_sidebar_scroll.log" |

### Severity Classification

| Severity | Criteria |
|----------|----------|
| "高" (High) | Data loss, crash, AI call failure, incorrect content generation |
| "中" (Mid) | Noticeable UX friction, missing expected feedback, suboptimal layout |
| "低" (Low) | Minor visual inconsistency, nice-to-have improvement, edge case polish |
