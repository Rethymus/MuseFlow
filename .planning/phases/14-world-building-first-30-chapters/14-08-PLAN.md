The fix: insert a pure-Dart helper `String enforceD11Bounds(String rawOutput)` between `adapter.createStream(...).join()` and `chapterRepository.updateDocumentContent(...)`. This helper:
- If `rawOutput.length > 500`: truncate at the last complete sentence boundary (period, exclamation, question mark, or newline) within the first 500 characters. If no sentence boundary found before 500, hard-truncate at 499 chars and append closing punctuation.
- If `rawOutput.length < 300` and `rawOutput.isNotEmpty`: accept as-is (log a warning) -- the lower bound is advisory; the real problem is exceeding 500.
- If `rawOutput.isEmpty`: throw an error (empty generation is a real failure per D-04).
- Return the processed string.

This is test-only code. It does NOT modify product code. The `enforceD11Bounds` function should be defined at the top of `serial_generation_test.dart` (or in `test/journey/helpers/` if shared).
