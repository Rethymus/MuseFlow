## Chart Specifications

### OperationTypePieChart

- Chart type: `PieChart` from fl_chart
- Height: 220px (consistent with existing charts)
- 5 sections, one per operation type
- Colors: use 5 distinct colors from Material 3 color scheme:
  - synthesis: `colorScheme.primary`
  - editorRewrite: `colorScheme.tertiary`
  - polish: `colorScheme.secondary`
  - freeInput: `colorScheme.primaryContainer`
  - deviationDetection: `colorScheme.tertiaryContainer`
- Each section shows operation type label and percentage
- Center radius: 44px (matches existing pie chart)
- Section radius: 58px
- Section spacing: 2px
- Empty state: SizedBox(220, Center(Text("还没有 Token 使用记录")))

### ChapterTokenBarChart

- Chart type: `BarChart` from fl_chart
- Height: 220px if <= 10 chapters; 220px with horizontal scroll for > 10
- Each bar represents one chapter
- Bar width: 16px
- Bar color: `colorScheme.primary`
- Y-axis: token count
- X-axis: chapter number labels
- Grid: `FlGridData(show: false)`
- Border: `FlBorderData(show: false)`
- Empty state: SizedBox(220, Center(Text("还没有章节级 Token 记录")))

### TokenTrendLineChart

- Chart type: `LineChart` from fl_chart
- Height: 220px
- Single line showing cumulative token usage over time
- Curved line, barWidth: 3, no dots
- Color: `colorScheme.primary`
- Grid: `FlGridData(show: false)`
- Border: `FlBorderData(show: false)`
- Empty state: SizedBox(220, Center(Text("还没有 Token 消耗趋势")))

## States

| State | Visual |
|-------|--------|
| Loading | `Center(child: CircularProgressIndicator())` |
| Empty (no records) | Inline Card with message: "开始使用 AI 功能后，这里会出现消耗统计。" |
| Error | Center column: "统计加载失败：{error}" + FilledButton "重试" |
| Data loaded | Full content: summary cards + 3 chart sections |
| Partial data (some charts empty) | Each chart handles its own empty state independently |

## Data Flow

```
TokenAuditNotifier (Riverpod AsyncNotifier)
  -> TokenAuditRepository (reads from Hive 'token_audit' box)
    -> List<TokenAuditRecord>

TokenAuditPage watches tokenAuditNotifierProvider
  -> .when(loading, error, data)

data flows to:
  1. TokenAuditSummary (aggregated totals)
  2. ChapterTokenBarChart (grouped by chapterId)
  3. OperationTypePieChart (grouped by operationType)
  4. TokenTrendLineChart (sorted by timestamp, cumulative)
```
