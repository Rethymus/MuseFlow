# Phase 16 -- UI Design Contract

> Visual and interaction contract for the Analysis & Reports phase.
> This phase builds four analytical report pages (REPORT-01 through REPORT-04)
> that aggregate data from Phases 12-15 and present it in a reports hub.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Flutter/Material 3, not React) |
| Preset | not applicable |
| Component library | Flutter Material 3 + fl_chart |
| Icon library | Material Icons (Icons.*) |
| Font | Noto Sans SC via google_fonts |

**Notes:** This is a Flutter/Dart project. shadcn gate does not apply. All UI follows existing Material 3 dark indigo theme defined in `lib/shared/theme/app_theme.dart`.

Inherited from Phase 15 UI-SPEC (unchanged).

| sm | 8px | Compact spacing, subtitle-to-value gap |

Exceptions:
- 12px for Wrap spacing between summary cards (established in WritingStatsPage)

**Source:** Inherited from Phase 15 UI-SPEC, originally from Phase 11 UI-SPEC.

## Color

Inherited from Phase 15 UI-SPEC (unchanged).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `colorScheme.surface` | Page background |
| Secondary (30%) | `colorScheme.surfaceContainerLow` | Summary cards, chart section cards, report hub cards |
| Accent (10%) | `colorScheme.primary` | Card icons, chart primary bars/lines, primary CTAs |
| Accent secondary | `colorScheme.tertiary` | Pie chart secondary segments, blind-read "human" verdict |
| Destructive | `colorScheme.error` | Error states, consistency flag "drift" severity |

Accent reserved for:
- StatsSummaryCard leading icons
- Primary bar chart series
- Line chart primary series
- "Export report" primary CTA buttons
- Report hub card leading icons
- Active/selected tab indicator
- Blind-read "AI generated" verdict button
- Consistency score positive indicator

Report-specific color usage:
- Cost projection chart: `colorScheme.primary` (actual) + `colorScheme.tertiary` (projected)
- Consistency drift chart: `colorScheme.primary` (consistent) + `colorScheme.error` (drift)
- Blind-read verdict: `colorScheme.primary` ("AI") + `colorScheme.tertiary` ("Human")
- Pain point severity: `colorScheme.error` (high) + `colorScheme.tertiary` (medium) + `colorScheme.onSurfaceVariant` (low)
- Report hub card status indicator: `colorScheme.primary` (has data) + `colorScheme.onSurfaceVariant` (no data)

**Source:** Inherited from Phase 15 UI-SPEC. Report-specific colors follow existing fl_chart patterns.

## Chart Specifications

### CostProjectionChart (REPORT-01)

- Chart type: `BarChart` from fl_chart
- Height: 220px
- 2 grouped bars: "实际" (actual) and "推算" (projected)
- Colors: `colorScheme.primary` (actual), `colorScheme.tertiary` (projected)
- Bar width: 48px per group
- Grid: `FlGridData(show: false)`
- Border: `FlBorderData(show: false)`
- X-axis labels: "输入 Token", "输出 Token", "API 调用"
- Empty state: not applicable (always shows projection if audit data exists)

### ConsistencyDriftChart (REPORT-04)

- Chart type: `LineChart` from fl_chart
- Height: 220px
- Single line showing consistency score per 10-chapter segment (10 segments for 100 chapters)
- Line color: `colorScheme.primary`
- Curved line, barWidth: 3, dot indicators at each point
- Y-axis: 0% to 100% consistency score
- X-axis: "1-10", "11-20", ..., "91-100"
- Grid: horizontal lines only, `FlGridData(show: true, drawVerticalLine: false)`
- Border: `FlBorderData(show: false)`
- Empty state: SizedBox(220, Center(Text("需要知识库实体和章节内容")))

## Report Export

All four reports support Markdown export following the existing `ExportService` file-writing pattern.

| Report | Export filename | Format |
|--------|----------------|--------|
| REPORT-01 | `token-cost-report.md` | Markdown with tables for summary, projection, suggestions |
| REPORT-02 | `pain-point-report.md` | Markdown with categorized issue list |
| REPORT-03 | `anti-ai-scent-report.md` | Markdown with verdict tally and interpretation |
| REPORT-04 | `consistency-report.md` | Markdown with entity scores and flag details |

**Export button placement:** Each report detail page has an export icon button in the AppBar (same pattern as manuscript export).

**Implementation:** Reuse `ExportService` file-writing abstraction. `ReportExportService` converts report data models to Markdown strings. Uses `path_provider` for directory and `share_plus` for sharing.
