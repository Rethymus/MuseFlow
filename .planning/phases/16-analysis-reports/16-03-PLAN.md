    Create `BlindReadService` in `lib/features/reports/application/blind_read_service.dart`. Constructor takes `ChapterRepository`. Methods:

    `selectExcerpts({String? manuscriptId, int count = 10, int minParagraphLength = 50})`:
    1. Get all chapters from repository (filter by manuscriptId if provided)
    2. For each chapter, extract paragraphs: split content by double-newline (`\n\n`), trim whitespace
    3. Filter paragraphs >= minParagraphLength characters
    4. Collect all eligible paragraphs into a list with their chapterId and chapterIndex
    5. If fewer than `count` paragraphs available, return all of them (no padding)
    6. Shuffle the list using `Random().shuffle()`
    7. Take first `count` excerpts
    8. Return List<BlindReadExcerpt> with humanVerdict=null (unjudged)

    `computeResult(List<BlindReadExcerpt> excerpts)`:
    1. Filter excerpts where humanVerdict != null (judged ones)
    2. totalJudged = judged.length
    3. correctCount = excerpts where humanVerdict == true (all content IS AI-generated per requirement, so true=correct)
    4. score = totalJudged > 0 ? correctCount / totalJudged : 0.0
    5. Return BlindReadResult with all fields

    Create `ConsistencyAnalysisService` in `lib/features/reports/application/consistency_analysis_service.dart`. Constructor takes `CharacterCardRepository`, `WorldSettingRepository`, `SkillRepository`, `ChapterRepository`, `NameIndexService`. Method:

    `analyze(String? manuscriptId)`:
    1. Load all chapters (filter by manuscriptId), sort by sortIndex
    2. Load all character cards, world settings, skill documents
    3. For each character card: build name list (name + aliases). Search each chapter content for name occurrences. Count chaptersWhereMentioned. Compute consistencyScore = chaptersWhereMentioned / totalChapters. Generate flags for chapters where entity is NOT mentioned (field="presence", expectedValue="mentioned", observedText="not found", severity=warning for first absence, critical for 5+ consecutive absences).
    4. For each world setting: extract key terms from name/rules/factions. Search chapters similarly. Generate flags for missing keywords in later chapters.
    5. For skills: check if skill document name appears in chapters.
    6. Compute driftPerSegment: divide 100 chapters into 10 segments of 10. For each segment, compute segment consistency = (total mentions in segment / (entities * 10)). Normalize to 0.0-1.0.
    7. overallConsistencyScore = average of all entity consistencyScores
    8. Return ConsistencyReport

    - `consistencyReportServiceProvider` -- Provider<ConsistencyAnalysisService>
