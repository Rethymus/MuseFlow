## FakeAdapter Output Contract

### Deterministic Response Content

FakeAdapter must return fixed, assertable text for each operation type. Content is Chinese xianxia genre prose.

| Operation Type | Response Index | Response Text (first 40 chars) | Full Length |
|----------------|---------------|-------------------------------|-------------|
| synthesis | 0 | "林风立于青云峰巅，剑气纵横三千里..." | ~40 chars |
| synthesis | 1 | "破晓时分，灵气如潮涌入丹田..." | ~40 chars |
| synthesis | 2 | "古洞深处，一枚玉简静静悬浮..." | ~40 chars |
| rewrite | 0 | "剑光一闪，血溅三尺..." | ~30 chars |
| rewrite | 1 | "灵力汇聚掌心，化作一道青色光柱..." | ~30 chars |
| polish | 0 | "他深吸一口气，缓缓运转..." | ~40 chars |
| polish | 1 | "月华如水，洒在剑身之上..." | ~30 chars |
| freeInput | 0 | "此剑名为'斩仙'，乃上古仙人遗留之物..." | ~35 chars |

### Assertion Patterns

Tests should assert against these known substrings:

| Operation | Assert Contains | Reason |
|-----------|----------------|--------|
| synthesis (call 1) | `'林风'` | Protagonist name appears in first synthesis response |
| synthesis (call 1) | `'筑基'` | Cultivation term confirms xianxia genre |
| rewrite | `'剑光'` | Sword imagery confirms rewrite tone |
| polish | `'灵力'` or `'月华'` | Cultivation aesthetic confirms polish output |
| Any operation | `isNotEmpty` | Minimum: response is non-empty string |

### Token Estimation

FakeAdapter uses simplified token estimation for test assertions:

| Input | Formula | Example |
|-------|---------|---------|
| Chinese text | `text.length * 2` | "林风" (2 chars) = 4 tokens |
| Mixed text | `nonWhitespace.length * 2` | "Hello 林风" (7 non-ws chars) = 14 tokens |

**Token audit assertions:**
- `record.inputTokens > 0` -- must have positive input tokens
- `record.outputTokens > 0` -- must have positive output tokens
- `record.totalTokens == record.inputTokens + record.outputTokens` -- sum invariant

## Integration Test Flow Contract

### TEST-02: UI Flow Sequence

The integration test drives this exact sequence through the existing UI:

```
1. Launch app with ProviderScope(overrides: [FakeAdapter])
   -> Land on ManuscriptLibraryPage (default route: /editor)
   -> Assert: find.text('文稿库') or find.text('创建你的第一部作品')

2. Create manuscript
   -> Tap FAB (find.byIcon(Icons.add))
   -> Enter title: find.byKey(Key('manuscript_title')) -> enterText('剑道苍穹')
   -> Enter genre: find.byKey(Key('manuscript_genre')) -> enterText('修仙')
   -> Tap confirm: find.text('确认')
   -> Assert: find.text('剑道苍穹') visible in grid

3. Enter manuscript editor
   -> Tap manuscript card: find.text('剑道苍穹')
   -> Navigate to /manuscript/{id}/editor
   -> Assert: editor view loaded

4. Create chapters (batch: 3-5 for UI test, not 100)
   -> Tap add chapter: find.byKey(Key('add_chapter_button'))
   -> Enter title: find.byKey(Key('chapter_title_field')) -> enterText('第一章')
   -> Confirm
   -> Repeat for chapters 2-5

5. AI generation (1 chapter only)
   -> Navigate to a chapter
   -> Tap AI synthesis: find.byKey(Key('ai_synthesis_button'))
   -> Wait for streaming: tester.pump(Duration(seconds: 2))
   -> Assert: find.textContaining('林风') in editor

6. Export
   -> Tap export: find.byKey(Key('export_button'))
   -> Select format: find.text('Markdown')
   -> Confirm export
   -> Assert: success indicator (SnackBar or log)
```
