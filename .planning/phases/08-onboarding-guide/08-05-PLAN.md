<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user→OpeningGeneratorService | Untrusted user input (storyConcept) sent to AI API |
| AI response→OpeningGeneratorService | Untrusted AI output (malformed JSON, excessive length) parsed and inserted into editor |
| editor→OpeningGeneratorSheet | Editor document state accessed for text insertion |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-10 | Tampering | Story concept input in OpeningGeneratorService | mitigate | Input length limit enforced (500 chars max, truncate), no raw user input in system prompt, structured JSON output parsing (already handled in Plan 08-04) |
| T-08-11 | Tampering | Editor insertion at wrong position corrupts document | mitigate | Use super_editor request system (InsertTextRequest, InsertNodeRequest) with proper selection handling, existing pattern proven in EditorAINotifier |
</threat_model>
