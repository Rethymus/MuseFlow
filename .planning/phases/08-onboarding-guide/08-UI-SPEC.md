## Spacing Scale

Declared values (multiples of 4, matching existing codebase):

| Token | Value | Usage | Codebase Evidence |
|-------|-------|-------|-------------------|
| xs | 4px | Icon gaps, inline padding, tag vertical padding | `_PassiveTag` vertical: 3px (approx); `SizedBox(width: 4)` in toolbar dividers |
| sm | 8px | Compact element spacing, toolbar padding, button gaps | `EdgeInsets.only(bottom: 12)` (rounded); toolbar `horizontal: 8, vertical: 4` |
| md | 16px | Default element spacing, page horizontal padding | `EdgeInsets.fromLTRB(16, 12, 16, 8)` in gallery; `EdgeInsets.symmetric(horizontal: 16)` in inputs |
| lg | 24px | Section padding, icon sizes | `Icon(icon)` at default 24px; `SizedBox(height: 24)` |
| xl | 32px | Layout gaps | Wizard step content vertical padding |
| 2xl | 48px | Major section breaks | Between wizard header and step content |
| 3xl | 64px | Page-level spacing | Wizard full-screen top margin |

Exceptions:
- 12px for card bottom margins (`EdgeInsets.only(bottom: 12)` in `_TemplateCard`) -- standard across gallery/list views
- Card/tag border radius: 12px for cards/inputs, 999px for pill tags (existing convention)

Inherited from Material 3 dark indigo seed:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `colorScheme.surface` (dark) | Wizard background, page surfaces, scaffold |
| Secondary (30%) | `colorScheme.surfaceContainerHighest` | Cards, step panels, bottom sheet background, toolbar |
| Accent (10%) | `colorScheme.primary` (indigo) | Active step indicator, selected opening card border, "next" button, progress dot |

Accent reserved for:
- Active step indicator dot and progress bar fill
- Selected opening variant card border/outline
- Primary CTA button (`FilledButton`)

## Interaction Contracts

### Wizard Navigation

| Interaction | Behavior |
|-------------|----------|
| Swipe left | Advance to next step (if current step valid or skippable) |
| Swipe right | Go to previous step |
| "下一步" button | Validate current step, save progress, advance with 300ms `easeInOut` animation |
| "跳过" button | Mark step as completed (no entity save), advance to next step |
| "上一步" button | Return to previous step (state preserved via `AutomaticKeepAliveClientMixin`) |
| Close/dismiss | Exit wizard entirely, save progress, set `onboarding_completed: true`, navigate to `/editor` |

### Step Validation

| Step | Valid When | Skip Behavior |
|------|-----------|---------------|
| 1. Genre | Template selected (non-null `templateId`) | Skip: no template selected, proceed without |
| 2. World | World name non-empty | Skip: no world created, proceed without |
| 3. Character | Character name non-empty | Skip: no character created, proceed without |
| 4. Opening | AI generation completed AND one variant selected | Skip: no opening inserted, proceed to editor |

### Progress Bar

- Horizontal linear progress indicator at top of wizard
- 4 segments, fills proportionally (0% / 25% / 50% / 75% / 100%)
- Completed steps shown as filled dots, current step as indigo dot, future steps as outline dots
- Animation: 300ms easeInOut on step transition

### Opening Generator (Bottom Sheet)

| Interaction | Behavior |
|-------------|----------|
| Toolbar button tap | Opens `showModalBottomSheet(isScrollControlled: true)` |
| Generate button | Calls `OpeningGeneratorService`, shows `CircularProgressIndicator` |
| Generation complete | Displays 3 `OpeningVariantCard` in vertical scroll |
| Card tap / select button | Inserts opening text at cursor position in editor, closes bottom sheet |
| Close sheet | No text inserted |

### Opening Variant Card

- `Card` with 12px border radius
- Top: style badge (pill tag, "场景切入" / "人物切入" / "悬念切入") in `colorScheme.surfaceContainerHighest`
- Body: opening text preview, max 4 lines, `TextOverflow.ellipsis`
- Bottom: "使用此开篇" `FilledButton.tonal` (unselected) or `FilledButton` (selected state)
- Selected state: card border changes to `colorScheme.primary`, 2px width

## State Preservation

| Concern | Solution |
|---------|----------|
| PageView page disposal losing form data | `AutomaticKeepAliveClientMixin` on each step page |
| Wizard interruption (app killed) | `OnboardingProgress` persisted to Hive `settings` box as `onboarding_progress` key |
| Resume on relaunch | go_router `redirect` reads `onboarding_progress`, resumes at `currentStep` |
| Form controllers across swipes | Parent `OnboardingWizardPage` holds `TextEditingController` instances, passes to step pages |

## Accessibility

| Concern | Solution |
|---------|----------|
| Wizard steps | Semantic labels on step indicators (`Semantics(label: 'Step 2 of 4: Create World')`) |
| Skip/close buttons | Tooltip and semantic label ("Skip this step") |
| Opening variant cards | `Semantics` wrapper with button role for selection |
| Progress indicator | `Semantics` with "Progress: 50%, Step 2 of 4" |
| Form validation errors | `SnackBar` with `Semantics` live region for screen readers |
| Color-only indicators | Progress dots use both color (indigo vs outline) and fill (solid vs empty) to indicate state |
