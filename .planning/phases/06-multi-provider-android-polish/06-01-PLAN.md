<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> ProviderService.testConnection | User-provided apiKey, baseUrl, and model cross into API call construction |
| ProviderService -> External API (Claude endpoint) | Network call to https://api.anthropic.com/v1/ with user's API key |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | testConnection model parameter | mitigate | Model parameter is used in API request construction. Value is user-provided from form field -- no injection risk since openai_dart sends it as a JSON string field, not interpolated into URL. Existing _validateBaseUrl enforces HTTPS. |
| T-06-02 | Information Disclosure | Claude API key in network traffic | mitigate | _validateBaseUrl already enforces HTTPS. API key sent via Authorization header over TLS. Key stored in flutter_secure_storage (Windows Credential Manager / Android Keystore). |
| T-06-03 | Spoofing | Claude preset baseUrl | accept | Preset baseUrl is hardcoded as https://api.anthropic.com/v1/ -- user can edit it for custom endpoints. No security impact from user editing their own provider URL. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this plan. All changes extend existing code. |
</threat_model>
