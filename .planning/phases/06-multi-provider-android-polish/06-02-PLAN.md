## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input (parameter fields) -> AIProvider entity | Numeric text input parsed and stored |
| ProviderManagementNotifier -> External /v1/models endpoint | Network call with user's API key |
| AIProvider entity -> ChatCompletionCreateRequest | Entity fields forwarded to API request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Tampering | Parameter input fields | mitigate | Numeric range validation: temperature 0-2, topP 0-1, maxTokens 1-128000. Non-numeric input rejected. Per RESEARCH Pitfall 2: empty string maps to null, not 0. |
| T-06-05 | Information Disclosure | API key in model list fetch | mitigate | fetchModelList uses HTTPS-only baseUrl (enforced by existing _validateBaseUrl). API key sent via Authorization header over TLS. |
| T-06-06 | Spoofing | Model list response tampering | accept | Model list is for user convenience -- user can always type manually. No security impact from a tampered list. Per D-08: silent fallback. |
| T-06-07 | Tampering | Malicious baseUrl in model fetch | mitigate | _validateBaseUrl already enforces HTTPS. fetchModelList reuses the same validation. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed. All changes extend existing code and use existing openai_dart APIs. |
</threat_model>
