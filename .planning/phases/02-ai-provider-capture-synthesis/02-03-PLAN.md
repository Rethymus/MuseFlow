    - SynthesisState fields: accumulatedText (String), isStreaming (bool), isEditing (bool), error (String?), excludedFragmentsNotice (String?), highlights (List<TextHighlight> from anti-AI-scent)
    - SynthesisNotifier.startSynthesis(): reads selectedFragmentsProvider, gets active provider, gets API key, builds prompt via PromptPipeline, calculates token budget, starts streaming. State transitions: idle -> isStreaming=true with empty accumulatedText
    - Stream handling: subscribes to OpenAIAdapter.createStream, appends each token to accumulatedText. State updates on each token for real-time UI
    - Stream completion: on stream done, runs AntiAIScentProcessor.process on accumulatedText, updates state with processedText and highlights. Sets isStreaming=false, isEditing=true
    - Stream interruption per D-15: on error, preserve accumulatedText, set error message "生成中断，可继续编辑或重试", set isStreaming=false, isEditing=true. User can still edit partial content
    - Error handling per D-14: error messages displayed inline (not SnackBar). "网络连接失败", "请求太快，请稍后再试", "API Key 无效，请检查设置". Each error includes a "重试" option via regenerate method
    - regenerate(String? additionalInstruction): resets state and calls startSynthesis with the optional追加指令 per D-06. Each generation is independent
    - confirmAndInsert(): inserts accumulatedText into editor at cursor via InsertPlainTextAtCaretRequest. Navigates to editor page per D-07
    - Token budget per D-13: when fragments exceed budget, use TokenBudgetCalculator.selectFragmentsWithinBudget. Set excludedFragmentsNotice to "已移除 N 个碎片以保证质量"
