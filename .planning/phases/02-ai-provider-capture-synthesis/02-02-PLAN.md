  <action>
    Create AntiAIScentProcessor in application layer. This is a pure Dart class with no external dependencies.

    The process method takes text (String) and bannedPhrases (List<String>), returns ProcessingResult. Processing has two phases:
    1. Auto-replacement phase: iterate over a fixed synonym map (seeded per D-09/D-10). For each entry, use boundary-aware regex replacement. Boundary awareness per Pitfall 5: check that the character before the match is one of: string start, whitespace, Chinese punctuation (。，！？；：""''（）【】《》...), or newline. Same for character after. This prevents replacing "然而" inside a compound word.
    2. Structural highlight phase: scan for structural patterns (套话句式) using predefined regex patterns. When found, wrap with 【】 markers and add to highlights list. Per D-10: complex structural patterns are highlighted for user to manually fix, not auto-replaced.

    The initial banned phrase seed list (Claude's discretion per CONTEXT.md):
    - Auto-replace map: {"然而": "但是", "综上所述": "", "值得注意的是": "", "毫无疑问": "", "不可否认": "", "首先": "", "其次": "", "最后": "", "总的来说": "", "一方面": "", "另一方面": "", "总之": "", "显而易见": "", "不言而喻": "", "众所周知": ""}
    - Structural pattern regexes (highlighted, not replaced): r"不仅[^，。！？\n]{1,20}而且", r"随着[^，。！？\n]{1,20}的发展", r"在[^，。！？\n]{1,20}中，[^，。！？\n]{1,20}发挥了重要作用"

    Create ProcessingResult class: processedText (String after replacements and highlight markers), highlights (List<TextHighlight>). TextHighlight has: start (int), end (int), originalText (String), type (HighlightType enum: bannedWord, structuralPattern).

    - calculateBudget: subtract system prompt + persona + banned list + reserved output from model context window. Return available tokens for fragments.
    - selectFragmentsWithinBudget: iterate fragments in selection order, estimate tokens for each, accumulate until budget exceeded. When exceeded, stop adding and return BudgetResult with included list and excludedCount. Per D-13: LIFO removal (last-selected fragments excluded first). Display notice: "已移除 N 个碎片以保证质量" handled by the caller (synthesis notifier in Plan 03).

    Create BudgetResult class: included (List<Fragment>), excludedCount (int), budgetUsed (int), budgetTotal (int).
