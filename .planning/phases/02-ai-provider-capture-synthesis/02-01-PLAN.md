<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI → ProviderService | User input (name, baseUrl, apiKey) crosses into application layer |
| ProviderService → SecureStorage | API keys stored in platform secure storage |
| ProviderService → OpenAI-compatible API | Test connection sends API key to external service |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | SecureStorage API keys | mitigate | API keys stored via flutter_secure_storage (Windows Credential Manager / Android Keystore), never in Hive or logs |
| T-02-02 | Tampering | Provider baseUrl input | mitigate | Validate baseUrl format (must start with https:// or http:// for localhost only). Reject non-localhost http:// to prevent MITM |
| T-02-03 | Spoofing | Test Connection request | accept | Test request sends minimal payload, response not stored. Low-value target |
| T-02-04 | Information Disclosure | debugPrint / logging | mitigate | Never log API keys. Filter keys from any error messages before display |
| T-02-SC | Tampering | Dart pub packages | accept | All packages verified via pubspec.lock resolution and pub cache (see RESEARCH.md Package Legitimacy Audit) |
</threat_model>
