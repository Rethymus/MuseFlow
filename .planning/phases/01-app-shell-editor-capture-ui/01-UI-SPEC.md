# Phase 1 -- UI Design Contract

> Visual and interaction contract for the MuseFlow app shell, rich text editor, and fragment capture UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Flutter Material 3 |
| Preset | `ColorScheme.fromSeed(seedColor: Colors.indigo, brightness: Brightness.dark)` |
| Component library | Material 3 (NavigationRail, AppBar, IconButton, PopupMenuButton, etc.) |
| Icon library | Material Icons (`Icons.*`) -- built into Flutter SDK |
| Font | Noto Sans SC (CJK body text via google_fonts) + system serif fallback for editor prose |

**Rationale:** Flutter Material 3 is the project's UI framework (not React/shadcn). The indigo dark theme is already established in `lib/main.dart`. Noto Sans SC provides excellent Chinese rendering across weights and is the standard CJK choice for Google Fonts. The editor prose area uses a serif-adjacent feel for the "notebook" metaphor but defaults to Noto Sans SC since custom serif fonts add significant bundle size.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, toolbar button inner padding |
| sm | 8px | Compact element spacing, toolbar row padding |
| md | 16px | Default element spacing, sidebar vertical padding, card margins |
| lg | 24px | Section padding, editor page outer padding, card internal padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, editor top margin |
| 3xl | 64px | Page-level spacing, empty state illustration padding |

Exceptions:
- **Sidebar width (extended):** NavigationRail self-sizes to ~256px Material 3 standard (not 240px exact). Do not force override. Accept Material 3 default.
- **Sidebar width (collapsed):** NavigationRail self-sizes to ~72px Material 3 standard (not 64px exact). Accept Material 3 default.
- **Editor max-width:** 800px as constrained by `BoxConstraints(maxWidth: 800)` -- matches existing `main.dart` pattern.
- **Touch targets on Android:** 48px minimum per Material 3 accessibility guidelines (not on 8-point scale, but required).

## Typography

| Role | Size | Weight | Line Height | Font Family | Usage |
|------|------|--------|-------------|-------------|-------|
| Body | 14px | w400 (regular) | 1.5 | Noto Sans SC | Fragment text, settings labels, general UI text |
| Label | 12px | w500 (medium) | 1.3 | Noto Sans SC | Sidebar nav labels, tag chips, toolbar labels, timestamps |
| Heading | 20px | w600 (semi-bold) | 1.2 | Noto Sans SC | Page titles, section headers in settings |
| Display | 28px | w700 (bold) | 1.2 | Noto Sans SC | App title in sidebar header ("灵韵") |

**Editor text (special case):**
The super_editor document renders user content at a separate, configurable style. Editor body text defaults to 16px Noto Sans SC at w400 with 1.75 line-height for comfortable reading of Chinese prose. Headings within the document follow super_editor's built-in heading styles (H1: 28px, H2: 24px, H3: 20px), all in Noto Sans SC.

**Font weights declared:** exactly 3 -- w400 (regular), w500 (medium), w600/w700 (semi-bold/bold). w600 and w700 share the "emphasized" role; only use w700 for the single display case (app title).

**Chinese typography notes:**
- Noto Sans SC covers Simplified Chinese fully across all declared weights.
- Line-height 1.5 for body text is the minimum comfortable reading height for Chinese characters, which are taller than Latin ascenders.
- Editor line-height 1.75 gives Chinese prose a more spacious, book-like feel.

## Color

Based on Material 3 `ColorScheme.fromSeed(seedColor: Colors.indigo, brightness: Brightness.dark)`. The seed generates a full palette. Key roles mapped to the generated ColorScheme:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `ColorScheme.surface` (dark surface, ~#1C1B1F area) | Main background, editor background, content area |
| Secondary (30%) | `ColorScheme.surfaceContainerHighest` (elevated surface, ~#2B2930 area) | Sidebar background, toolbar background, capture cards |
| Accent (10%) | `ColorScheme.primary` (indigo accent, ~#B4A5FF area) | **Reserved for:** active nav item indicator, formatting toolbar active states (bold on, heading active), primary CTA button, quick-capture save button, text cursor |
| Destructive | `ColorScheme.error` (red, ~#FFB4AB area) | **Reserved for:** delete fragment confirmation only |

Accent reserved for (explicit list -- not "all interactive elements"):
1. Active navigation item indicator (NavigationRail selected icon background)
2. Formatting toolbar toggle active state (bold ON, italic ON, heading type active)
3. Primary CTA button background (quick-capture "保存" button)
4. Floating cursor/caret color in editor
5. Selected fragment checkbox fill

**Semantic additions:**
- `ColorScheme.onSurface` (light text on dark surface) -- all body text, labels
- `ColorScheme.onSurfaceVariant` (muted text) -- timestamps, secondary info, unselected nav labels
- `ColorScheme.outline` (subtle border) -- dividers, card outlines, toolbar separators

## Layout Contracts

### App Shell Layout

```
+--------+------------------------------------------+
| Sidebar|         Content Area                      |
| (Nav)  |   (StatefulShellRoute branch)             |
|        |                                           |
|  灵韵   |   +-----------------------------------+   |
|        |   |  [Toolbar - Editor branch only]    |   |
| 捕捉器  |   +-----------------------------------+   |
| 编辑器  |   |                                   |   |
| 设置    |   |   Centered content (max 800px)    |   |
|        |   |                                   |   |
|        |   +-----------------------------------+   |
+--------+------------------------------------------+
```

- Sidebar: NavigationRail, extended when window width >= 1000px, collapsed when 600-1000px.
- Below 600px (Android phone portrait): switch to `NavigationBar` at bottom, no sidebar.
- Content area: `Expanded` child in Row, fills remaining width.
- Breakpoint constants defined in `lib/shared/constants/app_constants.dart`.

### Editor Page Layout

```
+-----------------------------------------------+
| [B] [I] [Hv] | [*] [#]                       |  <- Fixed toolbar
+-----------------------------------------------+
|                                                |
|     +-------------------------------+          |
|     |                               |          |  <- max-width 800px, centered
|     |  Prose content area           |          |
|     |  (super_editor)               |          |
|     |                               |          |
|     +-------------------------------+          |
|                                                |
+-----------------------------------------------+
```

- Toolbar: fixed at top, `Container` with horizontal padding 8px, vertical padding 4px.
- Toolbar buttons: 48px touch target, icon 24px.
- Heading selector: PopupMenuButton with 3 items ("标题 1", "标题 2", "标题 3").
- Divider: 1px height, `ColorScheme.outline`.
- Editor area: `SingleChildScrollView` wrapping `Center` wrapping `ConstrainedBox(maxWidth: 800)`.
- Editor padding: 24px all sides (lg token).
- Editor background: `ColorScheme.surface` (same as dominant).

### Capture Page Layout

```
+-----------------------------------------------+
| [  输入灵感碎片，按回车添加...          ] [Enter]|  <- Input field (always visible)
+-----------------------------------------------+
| [全部] [故事] [章节] [场景]                     |  <- Filter chips
+-----------------------------------------------+
| [x] 这是一段灵感文字的示例              故事 · 第一章|
|     2026-06-01 14:30                           |
| [ ] 另一段碎片内容                      章节 · 第一节|
|     2026-06-01 14:25                           |
| [x] 第三段灵感                         场景       |
|     2026-06-01 14:20                           |
+-----------------------------------------------+
```

- Input field: `TextField` with `hintText`, borderless or subtle outline, padding 16px.
- Filter row: `Wrap` of `FilterChip` widgets, padding 16px horizontal.
- Fragment list: `ListView.builder`, each item is a `FragmentCard`.
- Fragment card: `Card` with elevation 0, `surfaceContainerHighest` background.
  - Left: `Checkbox` for multi-select.
  - Center: fragment text (body style), tag chips below.
  - Right: timestamp (label style, `onSurfaceVariant` color).
- Empty state: centered text with illustration placeholder (see Copywriting).

### Quick-Capture Overlay

```
+-------------------------------+
| 快速捕捉                  [x] |
|                               |
| [输入你的灵感...           ]  |
|                               |
|              [保存] [取消]    |
+-------------------------------+
```

- Opens as `showDialog()` with `AlertDialog` shape.
- Width: 480px max, centered on screen.
- TextField: autofocus, multiline (max 3 lines).
- "保存" button: primary (accent color).
- "取消" button: text button (onSurface color).
- Trigger: Ctrl+Shift+N via Flutter `Shortcuts`/`Actions`.
- On Enter key in TextField: same as pressing "保存".
- After save: brief success feedback (snackbar or dialog auto-close), then dialog closes.

## Interaction Contracts

### Navigation Interactions

| Action | Trigger | Response | State Change |
|--------|---------|----------|-------------|
| Switch to Editor | Tap "编辑器" in sidebar | Content area shows EditorPage, preserves editor content | go_router branch index 1 |
| Switch to Capture | Tap "捕捉器" in sidebar | Content area shows CapturePage, preserves fragment list scroll | go_router branch index 0 |
| Switch to Settings | Tap "设置" in sidebar | Content area shows SettingsPage | go_router branch index 2 |
| Sidebar collapse | Window resize to < 1000px | NavigationRail `extended` toggles to false, labels hidden, icons only | Layout-driven |
| Sidebar expand | Window resize to >= 1000px | NavigationRail `extended` toggles to true, labels shown | Layout-driven |

### Editor Interactions

| Action | Trigger | Response | State Change |
|--------|---------|----------|-------------|
| Toggle bold | Click bold button or Ctrl+B | Selected text gets/removes bold attribution; button toggles active state | Composer attribution updated |
| Toggle italic | Click italic button or Ctrl+I | Selected text gets/removes italic attribution; button toggles active state | Composer attribution updated |
| Set heading | Click heading dropdown, select H1/H2/H3 | Current paragraph converts to heading level | Document node replaced |
| Set unordered list | Click list button | Current paragraph converts to unordered ListItemNode | Document node replaced |
| Set ordered list | Click numbered list button | Current paragraph converts to ordered ListItemNode | Document node replaced |
| Type Chinese text | IME input | super_editor handles composition via platform channel | Document nodes updated |

### Capture Interactions

| Action | Trigger | Response | State Change |
|--------|---------|----------|-------------|
| Add fragment | Type text + press Enter in input field | New fragment appears at top of list, input clears | Hive box updated |
| Multi-select fragments | Check checkbox on fragment cards | Checkbox fills, fragment highlighted | Selection set updated |
| Filter by tag | Tap filter chip | List shows only fragments with matching tag | Filter state updated |
| Tag a fragment | Tap tag area on fragment card | Inline tag editor appears (Phase 1: simple chip add) | Fragment tags updated in Hive |

### Quick-Capture Interactions

| Action | Trigger | Response | State Change |
|--------|---------|----------|-------------|
| Open overlay | Ctrl+Shift+N from any screen | Dialog appears with autofocus text field | -- |
| Save fragment | Click "保存" or press Enter | Fragment saved to default tag, dialog closes | Hive box updated |
| Cancel | Click "取消" or press Escape | Dialog closes, no save | -- |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Minimum touch target | 48px on all interactive elements (toolbar buttons, nav items, checkboxes) |

| Color contrast | Material 3 dark theme meets WCAG AA by default; verify accent-on-surface contrast |
| Keyboard navigation | Toolbar buttons are Focus-traversable; Tab cycles through toolbar then editor |
