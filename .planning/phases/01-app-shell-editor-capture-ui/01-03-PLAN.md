    **Fragment service (lib/core/application/fragment_service.dart):**

    Create FragmentService as a use case class in the Application layer. This orchestrates fragment operations between the presentation layer and the repository.

    Constructor takes FragmentRepository. Methods:
    - createFragment(String text, {List<String>? tags}) -> Fragment: delegates to repository.addFragment, returns the created fragment
    - listFragments() -> List<Fragment>: returns all fragments sorted by createdAt descending (newest first)
    - listFragmentsByTag(String tag) -> List<Fragment>: if tag is '全部' (the "All" filter), returns all fragments; otherwise delegates to repository.getFragmentsByTag
    - removeFragment(String id): delegates to repository.deleteFragment
    - updateFragmentTags(String id, List<String> tags): fetches fragment, creates updated copy with new tags and updatedAt = DateTime.now(), delegates to repository.updateFragment
    - getDefaultTags() -> List<String>: returns [FragmentTags.story, FragmentTags.chapter, FragmentTags.scene] for the filter chip row

    **Capture provider (lib/features/capture/presentation/capture_provider.dart):**

    Create capture state management using @riverpod:

    CaptureState freezed class (or simple class): fragments (List<Fragment>), selectedIds (Set<String>), activeFilter (String, default '全部'), isLoading (bool).

    CaptureNotifier extending AsyncNotifier<CaptureState>:
    - build(): initializes by loading fragments from FragmentService, returns CaptureState with loaded data
    - addFragment(String text, {List<String>? tags}): creates fragment, reloads list, clears input
    - deleteFragment(String id): removes fragment, reloads list, removes from selectedIds if present
    - toggleSelect(String id): adds/removes id from selectedIds
    - selectAll(): selects all visible fragment IDs
    - clearSelection(): empties selectedIds
    - setFilter(String tag): updates activeFilter, reloads filtered list
    - updateTags(String id, List<String> tags): updates fragment tags

    Providers:
    - captureProvider: AsyncNotifierProvider for CaptureNotifier
    - captureInputProvider: StateProvider<String> for the input field text
    - fragmentFilterProvider: StateProvider<String> with default '全部'
    - selectedFragmentsProvider: computed provider returning fragments where id is in selectedIds

    **Tests (test/features/capture/fragment_tag_test.dart):**

    Unit tests for FragmentService with a mock FragmentRepository:
    - Test createFragment stores text and optional tags
    - Test listFragments returns sorted by createdAt descending
    - Test listFragmentsByTag with '全部' returns all
    - Test listFragmentsByTag with specific tag returns filtered subset
    - Test updateFragmentTags sets updatedAt
    - Test removeFragment delegates correctly

    Use Hive test helper from test/helpers/hive_test_helper.dart for in-memory Hive testing.
